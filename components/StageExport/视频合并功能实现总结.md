## 🎯 功能特性

### 视频合并流程
1. **初始化阶段** (0-5%)
   - 检查可合并的视频片段
   - 准备合并环境

2. **下载阶段** (5-20%)
   - 下载所有视频片段
   - 创建 HTMLVideoElement 实例
   - 显示当前进度（镜头 X / 总数）

3. **准备阶段** (20-30%)
   - 创建 Canvas 画布
   - 初始化 MediaRecorder
   - 选择最佳编码格式

4. **合并阶段** (30-90%)
   - 依次播放每个视频片段
   - 将每一帧绘制到 Canvas 上
   - MediaRecorder 实时录制
   - 显示实时进度

5. **输出阶段** (90-100%)
   - 停止录制
   - 生成 Blob 对象
   - 触发浏览器下载

  - 模态框覆盖层
  - 圆形进度指示器

## 📊 技术细节

### 核心技术栈
- **Canvas API**: 用于绘制视频帧
- **MediaRecorder API**: 用于录制 Canvas 内容
- **HTMLVideoElement**: 用于播放视频片段
- **Blob API**: 用于生成最终的视频文件

### 格式选择逻辑
```typescript
// 按优先级尝试编码格式
1. video/mp4;codecs=avc1.42E01E,mp4a.40.2  // MP4 (H.264 + AAC)
2. video/mp4                                    // MP4 (基础)
3. video/webm;codecs=vp9                        // WebM (VP9)
4. video/webm;codecs=vp8                        // WebM (VP8)
5. video/webm                                   // WebM (基础)
```

### 支持的配置选项
- **分辨率**: 720p (1280x720), 1080p (1920x1080), 4K (3840x2160)
- **帧率**: 24, 30, 60 fps
- **质量**: 0.0 - 1.0（默认 0.8）
- **比特率**: 5 Mbps（固定）

### 视频绘制逻辑
```typescript
// 保持宽高比，居中显示
const scale = Math.min(canvasWidth / videoWidth, canvasHeight / videoHeight);
const x = (canvasWidth - videoWidth * scale) / 2;
const y = (canvasHeight - videoHeight * scale) / 2;
ctx.drawImage(video, x, y, videoWidth * scale, videoHeight * scale);
```

### 性能优化
- 按需创建 Canvas 和 Video 元素
- 使用 requestAnimationFrame 优化绘制
- MediaRecorder 使用时间片（100ms）减少内存占用
- 异步处理避免阻塞 UI
- 进度回调实时更新

## 🚀 使用方法

### 用户操作流程
1. 用户进入"成片与导出"页签
2. 确认所有镜头视频已生成（进度 100%）
3. 点击"Merge Videos (.mp4/.webm)"按钮
4. 等待合并完成（显示进度模态框）
5. 合成完成后自动下载合并后的视频
6. 根据浏览器支持，下载 MP4 或 WebM 格式

### 代码示例
```typescript
import { mergeVideos, MergeProgress } from '../../services/videoMergeService';

const blob = await mergeVideos(project, {
  resolution: '1080p',
  frameRate: 30,
  quality: 0.8
}, (progress: MergeProgress) => {
  console.log(`${progress.phase}: ${progress.progress}%`);
  console.log(`镜头 ${progress.currentShot} / ${progress.totalShots}`);
});

// 自动根据 MIME 类型设置文件扩展名
const mimeType = blob.type;
const extension = mimeType.includes('mp4') ? 'mp4' : 'webm';

// 下载 blob
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url;
a.download = `merged_video.${extension}`;
a.click();
```

## ⚠️ 注意事项

### 浏览器兼容性
- 需要现代浏览器支持 Canvas 和 MediaRecorder API
- **推荐浏览器**:
  - Chrome 52+（支持 MP4 录制）
  - Edge 79+（支持 MP4 录制）
  - Safari 14.1+（支持 MP4 录制）
  - Firefox 25+（支持 WebM 录制）

### 格式支持
- **MP4 格式**:
  - Chrome、Edge、Safari 新版本支持
  - 最佳兼容性，推荐使用
  - H.264 编码

- **WebM 格式**:
  - 所有现代浏览器都支持
  - VP9/VP8 编码
  - 高压缩率

### 性能考虑
- 合并时间取决于视频数量和长度
- 需要实时播放所有视频片段
- 建议视频总数不超过 50 个
- 单个视频建议不超过 30 秒

### 内存使用
- Canvas 和 Video 元素运行在浏览器内存中
- 大量视频可能导致内存不足
- 建议分批合并长视频
- 合并期间请勿切换标签页

### 当前限制
- **不支持音频**: 当前版本只合并视频，不包含音频
- **无转场效果**: 视频之间直接拼接，无过渡效果
- **需要实时播放**: 合并速度受视频总时长限制

### 优势
- ✅ 无需下载大型依赖（FFmpeg.wasm ~25MB）
- ✅ 无需担心 CDN 访问问题
- ✅ 更快的启动速度
- ✅ 更好的浏览器兼容性
- ✅ 纯原生 API，无第三方依赖

### 劣势
- ⚠️ 需要实时播放视频，合并速度较慢
- ⚠️ 当前不支持音频合并
- ⚠️ 无转场效果
- ⚠️ 编码质量受浏览器限制
