# 视频合并技术实现文档

## 技术方案

### 方案选择

#### 初始方案：FFmpeg.wasm
- **优点**：
  - 功能强大，支持各种视频处理
  - 支持音频合并
  - 支持转场效果
  - 输出质量高

- **缺点**：
  - 需要下载大型依赖（~25MB）
  - CDN 访问可能失败
  - 启动速度慢
  - 需要配置 COOP/COEP 头

- **问题**：
  - 在实际测试中遇到 CDN 加载失败
  - ERR_QUIC_PROTOCOL_ERROR 错误
  - 无法正常工作

#### 最终方案：Canvas + MediaRecorder API
- **优点**：
  - 无需第三方依赖
  - 启动速度快
  - 浏览器原生支持
  - 无 CDN 访问问题

- **缺点**：
  - 需要实时播放视频
  - 合并速度较慢
  - 当前不支持音频
  - 无转场效果

- **选择原因**：
  - 稳定性高，不会因为网络问题失败
  - 实现简单，维护成本低
  - 浏览器兼容性好

## 核心实现

### 1. 视频合并流程

```typescript
async mergeVideos(project, options, onProgress) {
  // 1. 检查可合并的视频片段
  const completedShots = project.shots.filter(shot => shot.interval?.videoUrl);
  
  // 2. 下载所有视频片段
  const videoElements = [];
  for (const shot of completedShots) {
    const video = await loadVideo(shot.interval.videoUrl);
    videoElements.push(video);
  }
  
  // 3. 创建 Canvas 画布
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');
  
  // 4. 初始化 MediaRecorder
  const stream = canvas.captureStream(frameRate);
  const mediaRecorder = new MediaRecorder(stream, {
    mimeType: selectBestMimeType(),
    videoBitsPerSecond: 5000000
  });
  
  // 5. 开始录制
  mediaRecorder.start(100);
  
  // 6. 依次播放并绘制每个视频
  for (const video of videoElements) {
    await video.play();
    drawVideoToCanvas(video, ctx, canvas);
    await video.onended;
  }
  
  // 7. 停止录制并返回 Blob
  mediaRecorder.stop();
  const blob = await getBlobFromRecorder(mediaRecorder);
  return blob;
}
```

### 2. 视频加载

```typescript
async function loadVideo(videoUrl: string): Promise<HTMLVideoElement> {
  const video = document.createElement('video');
  video.muted = true;
  video.playsInline = true;
  
  if (videoUrl.startsWith('data:video')) {
    video.src = videoUrl;
  } else {
    const url = await getImageUrl(videoUrl);
    video.src = url;
  }
  
  await new Promise<void>((resolve, reject) => {
    video.onloadedmetadata = () => resolve();
    video.onerror = () => reject(new Error('视频加载失败'));
    video.load();
  });
  
  return video;
}
```

### 3. 视频绘制

```typescript
function drawVideoToCanvas(
  video: HTMLVideoElement,
  ctx: CanvasRenderingContext2D,
  canvas: HTMLCanvasElement
) {
  const canvasWidth = canvas.width;
  const canvasHeight = canvas.height;
  const videoWidth = video.videoWidth;
  const videoHeight = video.videoHeight;
  
  // 计算缩放比例，保持宽高比
  const scale = Math.min(canvasWidth / videoWidth, canvasHeight / videoHeight);
  
  // 计算居中位置
  const x = (canvasWidth - videoWidth * scale) / 2;
  const y = (canvasHeight - videoHeight * scale) / 2;
  
  // 绘制黑色背景
  ctx.fillStyle = '#000000';
  ctx.fillRect(0, 0, canvasWidth, canvasHeight);
  
  // 绘制视频帧
  ctx.drawImage(video, x, y, videoWidth * scale, videoHeight * scale);
}
```

### 4. 格式选择

```typescript
function selectBestMimeType(): string {
  const mimeTypes = [
    'video/mp4;codecs=avc1.42E01E,mp4a.40.2',  // MP4 (H.264 + AAC)
    'video/mp4',                                    // MP4 (基础)
    'video/webm;codecs=vp9',                        // WebM (VP9)
    'video/webm;codecs=vp8',                        // WebM (VP8)
    'video/webm'                                    // WebM (基础)
  ];
  
  for (const mimeType of mimeTypes) {
    if (MediaRecorder.isTypeSupported(mimeType)) {
      console.log(`[VideoMergeService] 使用编码格式: ${mimeType}`);
      return mimeType;
    }
  }
  
  throw new Error('浏览器不支持任何视频录制格式');
}
```

### 5. 实时绘制循环

```typescript
async function playAndRecordVideo(
  video: HTMLVideoElement,
  ctx: CanvasRenderingContext2D,
  canvas: HTMLCanvasElement,
  onProgress?: (progress: number) => void
) {
  await video.play();
  
  const animate = () => {
    if (video.ended || video.paused) {
      return;
    }
    
    drawVideoToCanvas(video, ctx, canvas);
    
    if (onProgress) {
      const progress = (video.currentTime / video.duration) * 100;
      onProgress(progress);
    }
    
    requestAnimationFrame(animate);
  };
  
  animate();
  
  await new Promise<void>((resolve) => {
    video.onended = () => {
      video.pause();
      video.currentTime = 0;
      resolve();
    };
  });
}
```

## 性能优化

### 1. 使用 requestAnimationFrame
```typescript
// 优化前

  drawVideoToCanvas(video, ctx, canvas);

  if (!video.ended) {
    drawVideoToCanvas(video, ctx, canvas);
```

### 4. 用户取消
```typescript
class VideoMergeService {
  private isCancelled = false;
  
  async mergeVideos(...) {
    for (const video of videoElements) {
      if (this.isCancelled) {
        mediaRecorder.stop();
        throw new Error('用户取消了合并操作');
      }
      await playAndRecord(video);
    }
  }
  
  cancel() {
    this.isCancelled = true;
  }
}
```

## 局限性

### 当前限制

1. **不支持音频**
   - MediaRecorder 录制 Canvas 时不会包含音频
   - 需要额外处理音频轨道
   - 后续版本可以添加音频合并

2. **无转场效果**
   - 视频之间直接拼接
   - 可以通过 Canvas 实现淡入淡出
   - 需要额外的代码实现

3. **需要实时播放**
   - 合并速度受视频总时长限制
   - 10 分钟的视频需要 10 分钟合并
   - 无法加速处理

4. **编码质量受限**
   - 受浏览器编码器限制
   - 无法自定义编码参数
   - 质量可能不如 FFmpeg

## 总结

Canvas + MediaRecorder API 方案虽然功能上不如 FFmpeg.wasm 强大，但在实际应用中更加稳定可靠。对于当前的视频合并需求，这个方案已经足够满足要求。

未来可以根据用户反馈和需求，逐步添加更多功能，如音频合并、转场效果等。
