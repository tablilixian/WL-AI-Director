# 视频合并功能使用指南

## 功能说明

视频合并功能使用 Canvas + MediaRecorder API 在浏览器端将多个视频片段合并为一个完整的视频文件。这是一个真正的视频拼接功能，而不是简单的 ZIP 打包。

## 使用步骤

1. **确保有可合并的视频**
   - 在"导演工作台"中生成视频片段
   - 确保至少有一个镜头的视频已生成完成

2. **点击合并按钮**
   - 进入"成片与导出"页签
   - 点击 **"Merge Videos (.mp4/.webm)"** 按钮

3. **等待合并完成**
   - 合并过程会播放所有视频片段
   - 请勿关闭浏览器标签页
   - 进度会实时显示

4. **下载合并后的视频**
   - 合并完成后会自动下载
   - 文件名格式：`{项目标题}_merged.mp4` 或 `{项目标题}_merged.webm`
   - 系统会自动选择浏览器支持的格式
   - 这是一个完整的视频文件，可以直接播放

## 技术说明

### 纯前端实现
- 使用 Canvas API 绘制视频帧
- 使用 MediaRecorder API 录制合并后的视频
- 无需服务器端处理
- 支持所有现代浏览器

### 合并原理
1. 创建一个 Canvas 画布
2. 依次播放每个视频片段
3. 将每一帧绘制到 Canvas 上
4. 使用 MediaRecorder 录制 Canvas 的内容
5. 生成一个完整的视频文件

### 支持的视频格式
- 输入：MP4、WebM 等格式
- 输入：Base64 编码的视频
- 输入：本地存储的视频
- 输出：MP4 格式（H.264 编码，优先）或 WebM 格式（VP9/VP8 编码，备选）

### 合并选项
默认配置：
- 分辨率：1080p (1920x1080)
- 帧率：30fps
- 编码格式：H.264（MP4）或 VP9（WebM）
- 比特率：5Mbps

### 格式选择逻辑
系统会按以下顺序尝试选择输出格式：
1. MP4 (H.264 + AAC) - 最佳兼容性
2. MP4 (基础) - 广泛支持
3. WebM (VP9) - 高压缩率
4. WebM (VP8) - 兼容性较好
5. WebM (基础) - 最后备选

## 常见问题

### Q: 合并速度很慢
**A:** 正常现象：
- 需要实时播放所有视频片段
- 视频数量多：需要更多时间
- 视频文件大：处理时间更长

**优化建议：**
1. 确保浏览器有足够的内存
2. 关闭其他占用资源的标签页
3. 等待合并完成

### Q: 下载视频片段失败
**A:** 可能的原因：
- 视频数据损坏
- 本地存储空间不足
- 视频格式不支持

**解决方法：**
1. 检查视频是否完整生成
2. 清理浏览器缓存
3. 重新生成视频片段

### Q: 合并后的视频质量不佳
**A:** 可能的原因：
- 原始视频质量较低
- 合并参数设置不当
- 浏览器编码限制

**解决方法：**
1. 提高原始视频生成质量
2. 调整合并参数（分辨率、帧率）
3. 使用专业视频编辑软件后期处理

### Q: 为什么下载的是 WebM 格式而不是 MP4？
**A:** 格式选择取决于浏览器支持：
- 系统会优先尝试 MP4 格式
- 如果浏览器不支持 MP4 录制，会自动回退到 WebM
- 大部分现代浏览器都支持 MP4 录制

**如何获得 MP4 格式：**
1. 使用 Chrome、Edge 等支持 MP4 的浏览器
2. 检查浏览器版本是否过旧
3. 使用在线转换工具将 WebM 转为 MP4

## 注意事项

1. **浏览器兼容性**
   - 支持所有现代浏览器
   - 推荐使用 Chrome、Firefox、Edge 最新版本
   - Safari 可能存在兼容性问题

2. **性能要求**
   - 建议使用 8GB 以上内存
   - 处理大量视频时可能占用较多 CPU
   - 合并期间请勿切换标签页

3. **存储空间**
   - 确保浏览器有足够的临时存储空间
   - 确保本地磁盘有足够空间存储输出视频

4. **网络要求**
   - 不需要网络连接（如果视频已缓存）
   - 首次下载视频片段需要网络

5. **音频处理**
   - 当前版本不支持音频合并
   - 输出视频将没有音频
   - 后续版本会添加音频支持

## 高级功能

### 自定义合并参数
修改 `videoMergeService.ts` 中的配置：

```typescript
const DEFAULT_OPTIONS: MergeOptions = {
  resolution: '1080p',  // 720p | 1080p | 4K
  frameRate: 30,         // 24 | 30 | 60
  quality: 0.8           // 0.0 - 1.0
};
```

### 自定义输出格式
修改 MediaRecorder 的 mimeType：

```typescript
const mediaRecorder = new MediaRecorder(stream, {
  mimeType: 'video/webm;codecs=vp9',  // 或 'video/webm;codecs=vp8'
  videoBitsPerSecond: 5000000
});
```
