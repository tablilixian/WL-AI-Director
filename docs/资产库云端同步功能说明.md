# 资产库云端同步功能说明

**最后更新时间**: 2026-03-09  
**功能版本**: v1.0

---

## 📋 功能概述

资产库云端同步功能允许用户在不同设备之间共享角色、场景、道具等资产，实现跨设备协作和数据同步。

### 核心特性

- ✅ **云端存储**: 资产库数据自动同步到 Supabase 云端数据库
- ✅ **本地缓存**: 资产库数据同时存储在本地 IndexedDB，提升访问速度
- ✅ **自动同步**: 保存资产时自动同步到云端，打开资产库时自动从云端拉取
- ✅ **离线支持**: 离线时使用本地数据，在线时自动同步
- ✅ **数据安全**: 基于 Supabase 的 RLS（行级安全）策略，确保数据安全

## 📊 数据库结构

### asset_library 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键，自动生成 |
| user_id | UUID | 用户 ID，外键关联 profiles 表 |
| type | TEXT | 资产类型（character/scene/prop/turnaround） |
| name | TEXT | 资产名称 |
| project_id | UUID | 关联项目 ID（可为空） |
| project_name | TEXT | 关联项目名称（可为空） |
| data | JSONB | 资产完整数据 |
| created_at | TIMESTAMP WITH TIME ZONE | 创建时间 |
| updated_at | TIMESTAMP WITH TIME ZONE | 更新时间 |

### 索引

- `idx_asset_library_user_id`: 用户 ID 索引
- `idx_asset_library_type`: 资产类型索引
- `idx_asset_library_project_id`: 项目 ID 索引
- `idx_asset_library_updated_at`: 更新时间索引（降序）

### RLS 策略

- 用户只能查看自己的资产库
- 用户只能插入自己的资产库
- 用户只能更新自己的资产库
- 用户只能删除自己的资产库

## 🔧 技术实现

### 1. API 层

**文件**: `src/api/assetLibrary.ts`

提供以下 API 方法：

- `list()`: 获取用户所有资产库项目
- `get(id)`: 获取单个资产库项目
- `create(item)`: 创建资产库项目
- `update(id, updates)`: 更新资产库项目
- `delete(id)`: 删除资产库项目
- `batchCreate(items)`: 批量创建资产库项目
- `listByType(type)`: 按类型筛选资产库项目

### 2. 存储服务层

**文件**: `services/hybridStorageService.ts`

实现混合存储策略：

- **读取策略**: 优先从云端获取，云端不可用时回退到本地
- **写入策略**: 双写策略，同时保存到云端和本地
- **删除策略**: 同时从云端和本地删除

#### getAllAssetLibraryItems()

```typescript
async getAllAssetLibraryItems(): Promise<AssetLibraryItem[]> {
  // 如果在线，优先从云端获取
  if (this.isOnline()) {
    try {
      const cloudItems = await assetLibraryApi.list();
      // 同步到本地 IndexedDB
      for (const item of cloudItems) {
        await saveAssetToLibraryToDB(item);
      }
      return cloudItems;
    } catch (error) {
      console.warn('[HybridStorage] 从云端获取素材库失败，回退到本地:', error);
    }
  }
  
  // 回退到本地 IndexedDB
  return getAllAssetLibraryItemsFromDB();
}
```

#### saveAssetToLibrary()

```typescript
async saveAssetToLibrary(item: AssetLibraryItem): Promise<void> {
  // 先保存到本地 IndexedDB
  await saveAssetToLibraryToDB(item);
  
  // 如果在线，同步到云端
  if (this.isOnline()) {
    await assetLibraryApi.create(item);
  }
}
```

### 3. 组件层

**文件**: `components/StageAssets/index.tsx`

组件通过 `hybridStorage` 服务调用资产库功能：

```typescript
import { getAllAssetLibraryItems, saveAssetToLibrary, deleteAssetFromLibrary } from '../../services/storageService';

// 获取资产库列表
const refreshLibrary = async () => {
  const items = await getAllAssetLibraryItems();
  setLibraryItems(items);
};

// 保存资产到库
const handleSaveToLibrary = async (item: AssetLibraryItem) => {
  await saveAssetToLibrary(item);
  refreshLibrary();
};
```

## 📝 使用说明

### 添加资产到库

在资产阶段（StageAssets）中：

1. 生成或上传角色/场景/道具
2. 点击"加入资产库"按钮
3. 资产自动保存到云端和本地

### 从资产库导入

1. 打开资产库模态框
2. 选择要导入的资产
3. 点击"导入"按钮
4. 资产自动添加到当前项目

### 替换资产

1. 在角色卡片上点击"从资产库替换"
2. 选择要替换的资产
3. 确认替换

### 删除资产

1. 打开资产库模态框
2. 点击要删除的资产
3. 点击"删除"按钮
4. 资产从云端和本地同时删除

## ⚠️ 注意事项

### 1. 数据一致性

- 资产库采用双写策略，同时保存到云端和本地
- 读取时优先从云端获取，确保数据最新
- 如果云端不可用，自动回退到本地数据

### 2. 离线使用

- 离线时只能使用本地数据
- 在线时会自动同步本地数据到云端
- 首次使用需要在线登录

### 3. 数据安全

- 基于 Supabase 的 RLS 策略，用户只能访问自己的数据
- 资产数据以 JSONB 格式存储在数据库中
- 建议定期备份重要资产

### 4. 性能优化

- 资产库数据会自动同步到本地 IndexedDB
- 本地数据访问速度更快
- 大量资产时建议分页加载

### 5. 图片存储

- 资产中的图片（参考图、九宫格等）以 Base64 格式存储在 JSONB 字段中
- Supabase JSONB 字段大小限制为 1MB
- 如果图片过大，建议使用 Supabase Storage 存储

## 🐛 常见问题

### Q1: 资产库列表为空？

**可能原因**:
- 未登录账号
- 网络连接失败
- 数据库表未创建

**解决方案**:
1. 检查是否已登录
2. 检查网络连接
3. 确认已执行 SQL 初始化脚本

### Q2: 资产没有同步到云端？

**可能原因**:
- API Key 配置错误
- Supabase 连接失败
- 权限不足

**解决方案**:
1. 检查 Supabase 配置
2. 查看控制台错误日志
3. 确认 RLS 策略正确配置

### Q3: 换设备后看不到资产？

**可能原因**:
- 使用了不同的账号登录
- 数据未同步到云端
- 网络连接问题

**解决方案**:
1. 确认使用同一账号登录
2. 检查云端数据是否存在
3. 检查网络连接

### Q4: 保存资产时报错？

**可能原因**:
- 数据格式错误
- 图片数据过大
- 数据库连接失败

**解决方案**:
1. 检查控制台错误日志
2. 确认数据格式正确
3. 减小图片尺寸
