# 资产库删除问题诊断指南

## 问题描述
从资产库删除资产后：
- ✅ 本地 IndexedDB 数据被删除
- ❌ 云端 Supabase 表格数据还在
- ❌ 云端存储（Storage）的图片还在
- ❌ 刷新页面后，资产库中还能看到已删除的资产

## 问题原因分析

### 可能的原因

1. **删除 API 调用失败**
   - Supabase RLS（行级安全策略）权限不足
   - 网络问题导致删除请求失败
   - 数据库权限配置问题

2. **删除逻辑不完整**
   - 只删除了本地 IndexedDB，未删除云端
   - 删除云端时出错，但错误被忽略

3. **数据同步问题**
   - 删除后没有刷新本地缓存
   - 刷新页面时从云端重新加载了已删除的数据

## 诊断步骤

### 步骤 1：检查控制台日志

打开浏览器开发者工具（F12），查看控制台输出：

```javascript

[AssetLibraryAPI] ✅ 删除成功：{id}
[Storage] ✅ 素材库项目已从云端删除
```

如果看到以下日志，说明有问题：
```javascript

```

### 步骤 2：检查 Supabase 控制台

1. 登录 [Supabase Dashboard](https://supabase.com/dashboard)
2. 进入你的项目
3. 打开 **Table Editor** → `asset_library` 表
4. 查看删除的资产记录是否还在

如果记录还在，检查：
- **RLS 策略**：是否有 DELETE 权限
- **触发器**：是否有阻止删除的触发器

### 步骤 3：检查 RLS 策略

在 Supabase SQL Editor 中运行：

```sql
-- 检查当前的 RLS 策略
SELECT * FROM pg_policies WHERE tablename = 'asset_library';

-- 检查是否有 DELETE 策略
SELECT schemaname, tablename, policyname, permissive, roles, cmd, qual, with_check
FROM pg_policies 
WHERE tablename = 'asset_library' AND cmd = 'DELETE';
```

**正确的 RLS 策略应该允许用户删除自己的数据：**

```sql
-- 启用 RLS
ALTER TABLE "public"."asset_library" ENABLE ROW LEVEL SECURITY;

-- 允许用户删除自己的资产
CREATE POLICY "允许用户删除自己的资产"
ON "public"."asset_library"
FOR DELETE
USING (auth.uid() = user_id);
```

如果数据库记录删除了，但图片还在：

1. 打开 Supabase **Storage** → `projects` bucket
2. 导航到：`{user_id}/asset_library/{type}/{id}/`
3. 检查图片文件是否还在

## 解决方案

### 方案 1：修复 RLS 策略（最可能）

在 Supabase SQL Editor 中运行：

```sql
-- 1. 检查当前策略
SELECT * FROM pg_policies WHERE tablename = 'asset_library';

-- 2. 如果缺少 DELETE 策略，添加它
DROP POLICY IF EXISTS "允许用户删除自己的资产" ON "public"."asset_library";

CREATE POLICY "允许用户删除自己的资产"
ON "public"."asset_library"
FOR DELETE
USING (auth.uid() = user_id);

-- 3. 确保也有 SELECT 策略（用于读取）
DROP POLICY IF EXISTS "允许用户查看自己的资产" ON "public"."asset_library";

CREATE POLICY "允许用户查看自己的资产"
ON "public"."asset_library"
FOR SELECT
USING (auth.uid() = user_id);
```

### 方案 2：添加错误处理和重试

修改删除逻辑，确保失败时抛出错误：

```typescript
// services/hybridStorageService.ts
async deleteAssetFromLibrary(id: string): Promise<void> {
  logger.debug(LogCategory.STORAGE, '删除素材库项目:', id);
  logger.debug(LogCategory.STORAGE, '当前在线状态:', this.isOnline());
  
  // 从本地 IndexedDB 删除
  await deleteAssetFromLibraryFromDB(id);
  logger.debug(LogCategory.STORAGE, '✅ 素材库项目已从本地 IndexedDB 删除');
  
  // 如果在线，从云端删除
  if (this.isOnline()) {
    try {
      await assetLibraryApi.delete(id);
      logger.debug(LogCategory.STORAGE, '✅ 素材库项目已从云端删除');
    } catch (error: any) {
      logger.error(LogCategory.STORAGE, '❌ 从云端删除失败:', error);
      throw error; // 抛出错误，让 UI 显示失败提示
    }
  }
}
```

## 常见问题

### Q1: 删除时提示 "permission denied for table asset_library"
**原因**：RLS 策略缺少 DELETE 权限
**解决**：添加 DELETE 策略（见方案 1）

### Q2: 删除成功但刷新后还在
**原因**：删除的是本地记录，云端未删除
**解决**：检查在线状态和 RLS 策略

### Q3: 数据库记录删除了，但图片还在
**原因**：删除逻辑没有清理云端存储
**解决**：修改删除逻辑，同时清理 Storage

### Q4: 控制台显示 "用户未在线"
**原因**：用户未登录或登录已过期
**解决**：重新登录后再删除
