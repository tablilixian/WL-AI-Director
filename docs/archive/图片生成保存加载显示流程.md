## 二、图片存储位置

### 1. IndexedDB - 本地浏览器数据库

#### WLDB 主数据库（services/dbConfig.ts）
```
数据库名: WLDB
版本: 7
```

| Store 名称 | 存储内容 | 关键字段 |
|-----------|---------|---------|
| `images` | 本地图片 Blob | `id`, `blob`, `createdAt`, `type`, `size` |
| `videos` | 本地视频 Blob | `id`, `blob`, `createdAt`, `type`, `size` |

**服务文件**: `services/imageStorageService.ts`

```typescript
// 核心 API
imageStorageService.saveImage(id: string, blob: Blob): Promise<void>
imageStorageService.getImage(id: string): Promise<Blob | null>
imageStorageService.deleteImage(id: string): Promise<void>
imageStorageService.uploadToCloud(id: string, blob: Blob, path: string): Promise<string>
```

#### Canvas Asset Store（src/modules/canvas/services/assetStore.ts）
```
数据库名: wl-canvas-assets
版本: 1
```

| Store 名称 | 存储内容 | 关键字段 |
|-----------|---------|---------|
| `assets` | 画布图片 Blob | `id`, `blob`, `mimeType`, `size`, `createdAt` |

```typescript
// 核心 API
assetStore.storeAsset(base64: string): Promise<string>  // 返回 asset ID
assetStore.getAssetUrl(id: string): Promise<string | null>  // 返回 blob: URL
assetStore.getAssetBlob(id: string): Promise<Blob | null>
assetStore.getAssetBase64(id: string): Promise<string | null>
assetStore.deleteAsset(id: string): Promise<void>
```

### 2. Supabase 云端存储

- **Bucket**: `projects`
- **用途**: 长期保存用户上传/生成的图片，跨设备同步
- **API**: `supabase.storage.from('projects').upload()`, `.getPublicUrl()`

### 3. localStorage

- **Key**: `wl-canvas-backup`
- **用途**: 画布状态备份（仅存储 JSON，不存储图片 Blob）
- **内容**: `CanvasState` 序列化后的 JSON（layers 数组，每个 layer 的 src 可能是 base64 或 `local:{id}`）

## 三、图片在 JSON 中的引用格式

图片在 JSON 中通过字符串引用，主要有以下几种格式：

### 格式 1: `local:{id}` 本地引用
```
示例: "local:img_1738561200_abc123"
说明: 引用 IndexedDB (WLDB images store) 中存储的图片
```

### 格式 2: `video:{id}` 本地视频引用
```
示例: "video:vid_1738561200_xyz789"
说明: 引用 IndexedDB (WLDB videos store) 中存储的视频
```

### 格式 3: Base64 内联
```
示例: "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEA..."
说明: 图片数据直接嵌入 JSON（体积大，不推荐用于大图）
```

### 格式 4: Cloud URL 云端 URL
```
示例: "https://xyz.supabase.co/storage/v1/object/public/projects/..."
说明: Supabase Storage 的公开 URL
```

### 格式 5: 普通 HTTP/HTTPS URL
```
示例: "https://example.com/image.jpg"
说明: 外部图片 URL
```

## 四、代码中的解析与显示流程

### 1. 图片 URL 解析工具（utils/imageUtils.ts）

```typescript
interface ImageSource {
  type: 'local' | 'cloud' | 'base64' | 'video';
  url?: string;
  localImageId?: string;
  localVideoId?: string;
}

// 解析图片 URL 为结构化对象
parseImageUrl(imageUrl: string | undefined): ImageSource

// 根据类型转换为可显示的 URL
getImageUrl(imageUrl: string | undefined): Promise<string | null>
```

**解析逻辑**:
```
1. if imageUrl.startsWith('local:')     → type: 'local', localImageId: 去掉前缀的 ID
2. if imageUrl.startsWith('video:')     → type: 'video', localVideoId: 去掉前缀的 ID
3. if imageUrl.startsWith('data:image') → type: 'base64', url: 原字符串
4. if imageUrl.startsWith('data:video') → type: 'base64', url: 原字符串
5. 其他                                 → type: 'cloud', url: 原字符串
```

### 2. 画布图层显示（src/modules/canvas/components/CanvasLayer.tsx）

```typescript
// 解析 src 为可显示的 URL
async function resolveImageSrc(src: string): Promise<string> {
  if (!src) return '';
  
  // 直接支持的格式：data:, http://, https://, blob:
  if (src.startsWith('data:') || src.startsWith('http://') || 
      src.startsWith('https://') || src.startsWith('blob:')) {
    return src;
  }
  
  // local: 格式 → 从 IndexedDB 读取
  if (src.startsWith('local:')) {
    const localId = src.replace('local:', '');
    const blob = await imageStorageService.getImage(localId);
    if (blob) {
      return URL.createObjectURL(blob);
    }
  }
  
  return '';
}

// React 组件中使用 useEffect 加载
useEffect(() => {
  const resolve = async () => {
    if (layer.type === 'image' || layer.type === 'drawing') {
      const resolved = await resolveImageSrc(layer.src);
      setResolvedSrc(resolved);
    }
  };
  resolve();
}, [layer.src, layer.type]);

// 渲染
return <img src={resolvedSrc} alt={layer.title} className="w-full h-full object-contain" />;
```

### 3. 从 JSON 恢复画布（src/modules/canvas/services/canvasIntegrationService.ts）

加载项目时，CanvasState 从 localStorage 恢复，每个 Layer 的 src 可能是：
- base64 字符串 → 直接使用
- `local:{id}` → 调用 `imageStorageService.getImage(id)` 获取 Blob
- URL → 直接使用

```
│                           图片生成                                       │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼

                                    │

                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  3. JSON 文件引用                                                       │
│     字段: localImageId / referenceImageSource                          │

                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  4. 可选：上传到 Supabase 云端                                          │
│     字段: referenceImage = 云端 URL                                    │

└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  5. 反序列化显示流程                                                   │
│                                                                         │

│       ↓                                                                 │
│  ImageSource.type === 'local' → imageStorageService.getImage(id)     │
│       ↓                                                                 │
│  Blob → URL.createObjectURL() → blob:http://...                       │
│       ↓                                                                 │
│  <img src="blob:http://..." /> → 浏览器渲染                           │
└─────────────────────────────────────────────────────────────────────────┘
```
