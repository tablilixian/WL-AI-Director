## 一、需求分析

### 1.1 业务背景

当前 AI 漫剧制作流程采用**线性工作流**：

```
剧本 → 资产生成 → 导演工作台 → 视频生成 → 导出
```

**核心痛点**：

| 痛点 | 影响 | 严重程度 |
|------|------|----------|
| AI 生成结果不可控 | 导演只能"重新生成"，无法"修改" | ⭐⭐⭐⭐⭐ |
| 批量调整效率低 | 逐张修改，耗时耗力 | ⭐⭐⭐⭐ |
| 风格一致性难保证 | 多次生成导致风格漂移 | ⭐⭐⭐⭐ |
| 工作流断裂 | 修改后难以回到主流程 | ⭐⭐⭐ |

### 1.2 用户画像

| 角色 | 需求 | 使用频率 |
|------|------|----------|
| **导演** | 精细化调整画面，确保视觉效果符合预期 | 高频 |
| **美术** | 批量处理角色/场景图片，统一风格 | 中频 |
| **制作人** | 审核和批准画面质量 | 低频 |

### 1.3 存储架构设计理念

#### 核心原则

```
资产库 = 成品资产库
├── 只保存最终确定可用的资产
├── 保存在云端（Supabase）
└── 每个项目有一条云端数据记录

临时资产 = 过程中的中间产物
├── 包括：生成的中间图片、画布编辑结果、版本对比图等
├── 保存在本地（IndexedDB）
└── 通过 JSON 文件管理引用关系

JSON 项目数据 = 项目结构和引用
├── 包括：剧本、分镜、资产引用、画布状态等
├── 保存在云端
└── 每个项目一条记录
```

#### 存储分层

| 层级 | 内容 | 存储位置 | 同步策略 |
|------|------|----------|----------|
| **资产库层** | 最终确定的角色定妆照、场景概念图 | 云端 (Supabase) | 实时同步 |
| **项目数据层** | 剧本、分镜、资产引用、画布状态 | 云端 (Supabase) | 保存时同步 |
| **临时资产层** | 生成的中间图片、编辑结果、版本对比 | 本地 (IndexedDB) | 不同步 |
| **缓存层** | 资产库的本地缓存 | 本地 (IndexedDB) | 按需同步 |

#### 数据流向

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              数据流向                                        │
└─────────────────────────────────────────────────────────────────────────────┘

AI 生成
    │
    ├──▶ 临时资产（本地 IndexedDB）
    │    • 生成的中间图片
    │    • 画布编辑结果
    │    • 版本对比图
    │
    └──▶ 确认后上传到资产库（云端 Supabase）
         • 最终确定的角色定妆照
         • 最终确定的场景概念图
         • 最终确定的关键帧

JSON 项目数据（云端 Supabase）
    │
    ├──▶ 剧本数据
    ├──▶ 分镜数据
    ├──▶ 资产引用（指向云端资产库）
    └──▶ 画布状态（图层位置、大小等）
```

#### 存储优势

| 优势 | 说明 |
|------|------|
| **降低云端成本** | 只上传最终资产，减少存储和带宽成本 |
| **提高本地性能** | 临时资产不用上传，本地操作更快 |
| **简化同步逻辑** | 只同步 JSON 和最终资产，减少冲突 |
| **支持离线工作** | 临时资产在本地，可以离线编辑 |

### 1.4 需求清单

#### 核心流程说明

```
重要调整（v1.1）：
───────────────────────────────────────────────────────────────────────────────
1. 导入来源：从本地定妆照/概念图导入，而不是从资产库
2. 回填目标：先替换本地数据，用户手动选择是否导出到资产库
3. 资产库定位：只存储用户确认的最终版本（云端）
4. JSON 引用：支持本地和云端两种格式
5. 文件格式：支持原始格式（png, mp4, jpg 等）
```

#### 数据流向

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              数据流向（调整后）                              │
└─────────────────────────────────────────────────────────────────────────────┘

AI 生成
    │
    ▼
本地数据（IndexedDB）
├── 定妆照（角色图片）
├── 概念图（场景图片）

    ▼

    ▼

```

#### JSON 引用格式

```typescript
// 支持本地和云端两种引用格式
interface AssetReference {
  id: string;
  name: string;
  type: 'character' | 'scene' | 'keyframe';
  
  // 本地引用（必填）
  localRef: string;              // "local:img_xxx" 格式
  
  // 云端引用（可选，用户导出到资产库后才有）
  cloudUrl?: string;             // "https://xxx.supabase.co/storage/..." 格式
  
  // 文件信息
  format: 'png' | 'jpg' | 'mp4' | 'webp';  // 原始格式
  size?: number;                 // 文件大小（字节）
  width?: number;                // 图片宽度
  height?: number;               // 图片高度
}
```

### 2.3 数据流转详细设计

#### 场景 A：角色定妆照编辑

```
Step 1: 用户操作
───────────────────────────────────────────────────────────────────────────────
  用户在"角色与场景"页签，查看生成的定妆照
  发现表情不够生动，想要调整
  点击"进入画布编辑"按钮

Step 2: 系统处理 - 导入
───────────────────────────────────────────────────────────────────────────────
  前端：
    1. 调用 canvasIntegrationService.importCharacterToCanvas()
    2. 传递参数：characterId, characterName, imageUrl
    3. 创建 LayerData 对象，设置 linkedResourceId = characterId
  
  数据结构：
    {
      id: "layer_xxx",
      type: "image",
      src: "local:img_xxx",
      imageId: "img_xxx",
      title: "主角定妆照",
      linkedResourceId: "character_001",
      linkedResourceType: "character"
    }

Step 3: 用户编辑
───────────────────────────────────────────────────────────────────────────────
  1. 创建 Prompt Layer
     - 模式：image-to-image
     - 提示词："Make the expression more intense and dramatic"
     - 点击"✨ 增强"
  
  2. 关联图片
     - 将定妆照关联到 Prompt Layer
  
  3. 执行生成
     - 点击"▶️ 执行"
     - 生成 3 个版本供选择

Step 4: 系统处理 - 回填
───────────────────────────────────────────────────────────────────────────────
  前端：
    1. 用户选择满意的版本
    2. 点击"导出回资产库"按钮
    3. 调用 exportCharacterFromCanvas(layerId)
    4. 获取修改后的图片 src
    5. 调用 updateCharacterImage(characterId, newImageUrl)
    6. 更新资产库中的定妆照
  
  后端：
    1. 如果是 base64，保存到 IndexedDB
    2. 更新 character 的 imageUrl 字段
    3. 保存修改历史（可选）

Step 5: 返回主流程
───────────────────────────────────────────────────────────────────────────────
  1. 关闭画布
  2. 返回"角色与场景"页签
  3. 定妆照已更新为修改后的版本
  4. 可以继续下一步"导演工作台"
```

#### 场景 B：首尾帧编辑

```
Step 1: 用户操作
───────────────────────────────────────────────────────────────────────────────
  用户在"导演工作台"，选择镜头的首尾帧
  发现构图不理想，想要调整
  点击"画布编辑"按钮

Step 2: 系统处理 - 导入
───────────────────────────────────────────────────────────────────────────────
  前端：
    1. 调用 canvasIntegrationService.importKeyframesToCanvas()
    2. 传递参数：keyframes = [startFrame, endFrame]
    3. 创建两个 LayerData 对象
    4. 设置 linkedResourceType = "keyframe"

Step 3: 用户编辑
───────────────────────────────────────────────────────────────────────────────
  1. 创建 Prompt Layer
     - 模式：image-to-image
     - 提示词："Adjust composition, center the character"
  
  2. 关联首尾帧
     - 将首帧和尾帧都关联到 Prompt Layer
  
  3. 执行生成
     - 生成修改后的首尾帧

Step 4: 系统处理 - 回填
───────────────────────────────────────────────────────────────────────────────
  前端：
    1. 用户选择满意的版本
    2. 点击"导出回导演工作台"按钮
    3. 调用 exportKeyframesFromCanvas(layerIds)
    4. 获取修改后的图片 src
    5. 调用 updateKeyframeImage(keyframeId, newImageUrl)
    6. 更新导演工作台中的首尾帧

Step 5: 返回主流程
───────────────────────────────────────────────────────────────────────────────
  1. 关闭画布
  2. 返回"导演工作台"
  3. 首尾帧已更新
  4. 可以继续"生成视频"
```

### 4.4 资产同步策略

**同步时机**：
1. 用户点击"导出回资产库"时 - 手动触发，画布 → 资产库
2. 用户点击"导出回导演工作台"时 - 手动触发，画布 → 导演工作台
3. 用户保存项目时 - 手动/自动，本地 → 云端

**同步规则**：
1. **冲突处理**：如果资产库中的图片已被修改（版本号变化），提示用户选择覆盖/保留两个版本/取消
2. **版本管理**：每次修改递增版本号，保留最近 5 个版本，支持回滚
3. **引用更新**：更新 assets.json 中的 imageUrl，更新 ProjectState 中的相关字段，触发 UI 刷新

## 六、技术难点分析

### 6.1 难点清单

| 难点 | 描述 | 复杂度 | 影响范围 |
|------|------|--------|----------|
| **双向同步** | 画布修改后如何同步回资产库 | ⭐⭐⭐⭐ | 全局 |
| **版本冲突** | 同一资产被多处修改时的冲突处理 | ⭐⭐⭐⭐ | 资产库 |
| **数据一致性** | 修改后如何保持引用关系正确 | ⭐⭐⭐ | 全局 |
| **性能优化** | 大量图片导入导出的性能 | ⭐⭐⭐ | 画布 |
| **用户体验** | 导入导出操作的直观性 | ⭐⭐⭐ | UI |

### 6.2 难点详细分析

#### 难点 1：双向同步

**问题**：画布中的图片被修改后，需要同步回资产库，但资产库中的图片可能同时被其他地方引用。

**解决方案**：
- **方案 A：立即同步（推荐）** - 用户点击"导出回资产库"时立即更新，所有引用该资产的地方自动刷新
- **方案 B：延迟同步** - 修改结果保存为"待定版本"，用户确认后再同步
- **方案 C：版本分支** - 修改结果创建新版本，用户可以选择使用哪个版本

**推荐方案**：方案 A + 版本管理 - 立即同步，但保留修改历史，支持回滚到任意版本

#### 难点 2：版本冲突

**问题**：同一资产可能被多处修改。

**解决方案**：
1. **版本号机制** - 每次修改递增版本号
2. **冲突检测** - 导出时检查版本号是否一致
3. **冲突解决选项** - 覆盖/保留/合并

#### 难点 3：数据一致性

**问题**：修改图片后，需要更新所有引用该图片的地方。

**解决方案**：
1. **引用追踪** - 记录每个资产被哪些地方引用
2. **级联更新** - 修改资产时，通知所有引用方
3. **更新策略** - 立即更新/延迟更新/可选更新

#### 难点 4：性能优化

**问题**：批量导入/导出大量图片时可能卡顿。

**解决方案**：
1. **懒加载** - 只加载可见区域的图片
2. **缩略图** - 生成缩略图用于预览
3. **分批处理** - 每次处理 3 张图片
4. **Web Worker** - 图片处理移至 Worker

#### 难点 5：用户体验

**问题**：导入导出操作需要直观易懂。

**解决方案**：
1. **清晰的文案** - "保存修改到角色库" 而不是 "导出回资产库"
2. **视觉反馈** - 修改后的图片添加"已修改"标记
3. **引导流程** - 首次使用时显示引导教程

  };
  
  addLayer(layer);
  return layerId;
}

/**
 * 将画布修改导出回资产库
 */
async exportToAssetLibrary(
  layerId: string
): Promise<{ success: boolean; message: string }> {
  const { layers } = useCanvasStore.getState();
  const layer = layers.find(l => l.id === layerId);
  
  if (!layer) {
    return { success: false, message: '图层不存在' };
  }
  
  if (!layer.linkedResourceId || !layer.linkedResourceType) {
    return { success: false, message: '图层未关联资产' };
  }
  
  // 保存新图片到 IndexedDB
  let newImageId: string;
  let newImageUrl: string;
  
  if (layer.src.startsWith('data:')) {
    newImageId = `canvas_export_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
    const response = await fetch(layer.src);
    const blob = await response.blob();
    await imageStorageService.saveImage(newImageId, blob);
    newImageUrl = `local:${newImageId}`;
  } else if (layer.src.startsWith('local:')) {
    newImageId = layer.imageId || layer.src.replace('local:', '');
    newImageUrl = layer.src;
  } else {
    return { success: false, message: '不支持的图片格式' };
  }
  
  // 更新资产库
  try {
    if (layer.linkedResourceType === 'character') {
      await hybridStorageService.updateCharacter(layer.linkedResourceId, {
        imageUrl: newImageUrl,
        imageId: newImageId,
        version: (layer.version || 1) + 1,
        updatedAt: Date.now()
      });
    }
    // ... 其他类型类似
    
    return { success: true, message: '导出成功' };
  } catch (error) {
    return { success: false, message: '导出失败：' + error.message };
  }
}
