# WL-AI-Director 存储架构文档

## 📋 概述

WL-AI-Director 采用**多层混合存储架构**，结合了本地存储（IndexedDB、localStorage）和云端存储（Supabase），提供离线工作能力和多设备同步功能。

### 存储层次

```
┌─────────────────────────────────────────────────────────────┐
│                    用户界面层                          │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│              Zustand (内存状态)                         │
│         - authStore (user, session, profile)             │
└─────────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┴─────────────────┐
        │                                   │
        ▼                                   ▼
┌─────────────────────┐           ┌─────────────────────┐
│   localStorage     │           │   IndexedDB (WLDB) │
│ - 模型配置        │           │ - projects         │
│ - API Key        │           │ - assetLibrary     │
│ - 主题设置        │           │ - images          │
│ - 语言设置        │           │ - videos          │
└─────────────────────┘           │ - projectStages    │
        │                       └─────────────────────┘
        │                                   │
        └─────────────────┬─────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│              Supabase (云端)                          │
│  - projects 表                                       │
│  - profiles 表                                       │
│  - asset_library 表                                   │
│  - Storage (图片/视频文件)                            │
└─────────────────────────────────────────────────────────────┘
```

## 1️⃣ IndexedDB (本地数据库)

### 基本信息

- **数据库名**: `WLDB`
- **当前版本**: `7`
- **配置文件**: `services/dbConfig.ts`

### 数据表结构

#### 1.1 `projects` - 项目数据表

**用途**: 存储所有项目的基本信息和完整状态

**主键**: `id` (字符串)

**数据结构**:
```typescript
interface ProjectState {
  id: string;                    // 项目唯一标识符
  title: string;                 // 项目标题
  createdAt: number;             // 创建时间戳
  lastModified: number;           // 最后修改时间戳
  version: number;                // 数据版本号，用于并发控制和冲突检测
  stage: 'script' | 'assets' | 'director' | 'export' | 'prompts';  // 当前阶段
  
  // 剧本阶段数据
  rawScript: string;             // 原始剧本文本
  targetDuration: string;         // 目标时长
  language: string;               // 语言
  visualStyle: string;            // 视觉风格：live-action, anime, 3d-animation 等
  shotGenerationModel: string;     // 分镜生成使用的模型
  scriptData: ScriptData | null;  // 解析后的剧本数据
  shots: Shot[];                // 镜头列表
  isParsingScript: boolean;       // 是否正在解析剧本
  renderLogs: RenderLog[];       // 渲染日志历史
}
```

**关键特性**:
- 所有媒体资源（图片、视频）以 Base64 格式存储在项目对象内部
- 删除项目时自动清理所有关联资源
- 支持版本号控制，防止并发冲突

#### 1.2 `assetLibrary` - 资产库数据表

**用途**: 存储可复用的角色、场景、道具和角色变体

**主键**: `id` (字符串)

**数据结构**:
```typescript
interface AssetLibraryItem {
  id: string;
  type: 'character' | 'scene' | 'prop' | 'turnaround';  // 资产类型
  name: string;
  projectId?: string;            // 关联的项目ID
  projectName?: string;          // 关联的项目名称
  createdAt: number;
  updatedAt: number;
  data: Character | Scene | Prop;  // 具体的资产数据
}
```

**资产类型**:
- **character**: 角色资产（包含变体、九宫格造型）
- **scene**: 场景资产
- **prop**: 道具资产
- **turnaround**: 角色九宫格造型设计

#### 1.3 `images` - 图片存储表

**用途**: 存储所有本地图片的二进制数据

**主键**: `id` (字符串)

**索引**: `createdAt` (非唯一)

**数据结构**:
```typescript
interface LocalImage {
  id: string;          // 图片唯一标识符
  blob: Blob;          // 实际的图片二进制数据
  createdAt: number;    // 创建时间戳
  type: string;         // MIME类型，如 'image/jpeg', 'image/png'
  size: number;         // 文件大小（字节）
}
```

**图片来源**:
- 角色参考图 (`local:{imageId}`)
- 场景参考图 (`local:{imageId}`)
- 道具参考图 (`local:{imageId}`)
- 角色变体图 (`local:{imageId}`)
- 关键帧图片 (`local:{imageId}`)
- 九宫格图片 (`local:{imageId}`)

**特性**:
- 支持自动清理过期图片（默认 7 天）
- 提供离线访问能力
- 避免云端 URL 过期问题

#### 1.4 `videos` - 视频存储表

**用途**: 存储所有本地视频的二进制数据

**主键**: `id` (字符串)

**索引**: `createdAt` (非唯一)

**数据结构**:
```typescript
interface LocalVideo {
  id: string;          // 视频唯一标识符
  blob: Blob;          // 实际的视频二进制数据
  createdAt: number;    // 创建时间戳
  type: string;         // MIME类型，如 'video/mp4'
  size: number;         // 文件大小（字节）
}
```

**视频来源**:
- 镜头视频片段 (`local:{videoId}`)
- 角色九宫格视频（如果有）

**特性**:
- 支持自动清理过期视频（默认 7 天）
- 提供离线访问能力

#### 1.5 `projectStages` - 项目阶段表

**用途**: 存储项目的阶段状态，避免频繁云端同步

**主键**: `projectId` (字符串)

**数据结构**:
```typescript
interface ProjectStage {
  projectId: string;     // 关联的项目ID
  stage: string;         // 阶段：'script' | 'assets' | 'director' | 'export' | 'prompts'
  updatedAt: number;     // 更新时间戳
}
```

**阶段说明**:
- `script`: 剧本编写阶段
- `assets`: 资产创建阶段
- `director`: 分镜导演阶段
- `export`: 导出阶段
- `prompts`: 提示词生成阶段

**设计目的**:
- 阶段变更频繁，单独存储避免触发完整项目同步
- 提升性能，减少云端请求

### 操作服务

| 服务文件 | 功能 |
|---------|------|
| `services/storageService.ts` | 项目和资产库的 CRUD 操作 |
| `services/imageStorageService.ts` | 图片和视频的存储操作 |
| `services/hybridStorageService.ts` | 混合存储策略（本地 + 云端） |

## 3️⃣ Supabase (云端存储)

### 数据库表

#### 3.1 `projects` 表

**用途**: 存储云端项目数据

**字段**:
```typescript
interface CloudProject {
  id: string;                    // 项目 UUID
  user_id: string;               // 用户 ID
  title: string;                 // 项目标题
  description?: string;          // 项目描述（原始剧本前 500 字符）

  created_at: string;           // 创建时间 (ISO 8601)
```

**特性**:
- 使用 `upsert` 操作，支持更新和插入

**字段**:

**用途**: 存储云端资产库

## 5️⃣ 混合存储策略

### 服务文件

**定义位置**: `services/hybridStorageService.ts`

### 核心策略

#### 5.1 读取策略

```
优先级: Supabase (云端) → IndexedDB (本地)
```

**流程**:
1. 检查用户登录状态
2. 如果已登录，尝试从 Supabase 读取
3. 如果 Supabase 失败或未登录，回退到 IndexedDB
4. 将云端数据静默同步到本地（异步，不阻塞）

#### 5.2 写入策略

```
双写: Supabase (云端) + IndexedDB (本地)
```

**流程**:
1. 先保存到 IndexedDB（保证离线可用）
2. 如果已登录，同时保存到 Supabase
3. 使用乐观锁防止并发冲突
4. 云端失败不影响本地保存

#### 5.3 同步策略

**登录时双向同步**:

```
本地新项目 → 推送到云端
云端新项目 → 恢复到本地
冲突项目 → 根据 version 和 lastModified 取舍
```

**冲突解决规则**:
- 版本号高的优先
- 版本号相同，lastModified 新的优先
- 记录冲突数量

#### 5.4 乐观锁机制

**目的**: 防止并发修改冲突

**实现**:
```typescript
interface Lock {
  projectId: string;
  locked: boolean;
  timestamp: number;    // 锁定时间
  version: number;     // 项目版本号
}
```

**规则**:
- 锁过期时间: 10 秒
- 版本号匹配才能获取锁
- 操作完成后释放锁

## 6️⃣ 数据流向详解

### 6.1 项目创建流程

```
用户创建项目
    ↓
Zustand (内存)
    ↓
IndexedDB (本地) ← 优先保存
    ↓
Supabase (云端) ← 异步同步（如果已登录）
```

### 6.2 项目读取流程

```
用户打开项目
    ↓
检查登录状态
    ↓
已登录 → Supabase (云端)
    ↓
失败/未登录 → IndexedDB (本地)
    ↓
Zustand (内存)
```

### 6.3 图片上传流程

```
用户上传图片
    ↓
IndexedDB (本地) ← 存储 Blob
    ↓
生成图片 ID
    ↓
Supabase Storage (云端) ← 异步上传
    ↓
返回公共 URL
    ↓
更新项目数据（引用 URL）
```

### 6.4 登录同步流程

```
用户登录
    ↓
authStore 更新状态
    ↓
触发双向同步
    ├─ 本地 → 云端 (上传新项目)
    ├─ 云端 → 本地 (下载新项目)
    └─ 冲突解决 (版本号 + 时间戳)
    ↓
静默同步资产库
    ↓
触发 UI 刷新
```

## 8️⃣ 数据持久化总结

| 数据类型 | 存储位置 | 持久化方式 | 同步策略 |
|---------|----------|------------|----------|
| 项目数据 | IndexedDB + Supabase | 双写 | 登录时双向同步 |
| 资产库 | IndexedDB + Supabase | 双写 | 登录时同步 |
| 图片/视频 | IndexedDB (Blob) + Supabase Storage (URL) | 本地 Blob，云端 URL | 异步上传 |
| 模型配置 | localStorage | 键值对 | 不同步 |
| API Key | localStorage | 键值对 | 不同步 |
| 主题设置 | localStorage | 键值对 | 不同步 |
| 语言设置 | localStorage | 键值对 | 不同步 |
| 用户认证 | Supabase + Zustand | 不持久化 | 每次刷新重新加载 |

## 9️⃣ 最佳实践

### 9.1 数据迁移

**IndexedDB 版本升级**:
1. 递增 `DB_VERSION`
2. 在 `onupgradeneeded` 中处理迁移逻辑
3. 自动检测并创建缺失的 store

**数据格式迁移**:
- 在 `loadRegistry()` 中处理旧格式数据
- 迁移完成后立即回写，避免重复执行
- 记录迁移日志

### 9.2 性能优化

**图片/视频存储**:
- 优先使用本地 IndexedDB（快速访问）
- 异步上传到云端（不阻塞 UI）
- 自动清理过期数据（默认 7 天）

**项目同步**:
- 使用 `projectStages` 避免频繁同步
- 乐观锁防止并发冲突
- 版本号控制冲突解决

### 9.3 错误处理

**网络错误**:
- Supabase 失败自动回退到 IndexedDB
- 本地操作不受网络影响
- 记录错误日志

**数据冲突**:
- 版本号高的优先
- 记录冲突数量
- 用户可手动解决

### 9.4 安全性

**API Key 管理**:
- 不暴露在前端代码中
- 支持多层级（模型级、提供商级、全局级）
- localStorage 加密存储（可选）

**用户数据隔离**:
- Supabase 使用 `user_id` 过滤
- 云端数据按用户隔离
- 本地数据共享（多用户）

## 🔟 常见问题

### Q1: 为什么图片要存两份？

**A**: 
- **IndexedDB**: 提供离线访问，避免 URL 过期
- **Supabase Storage**: 支持多设备同步，节省本地空间

### Q2: localStorage 和 IndexedDB 有什么区别？

**A**:
- **localStorage**: 键值对，适合小数据（配置、API Key）
- **IndexedDB**: 结构化数据库，适合大数据（项目、图片、视频）

### Q3: 为什么 authStore 不持久化？

**A**:
- 认证状态由 Supabase 管理
- 每次刷新重新加载，确保状态最新
- 使用 Session Token 自动续期

### Q4: 如何清理本地数据？

**A**:
- **清理过期图片/视频**: 调用 `imageStorageService.cleanOldImages()`
- **删除项目**: 自动清理所有关联资源
- **清空数据库**: 在开发者工具中手动清理 IndexedDB

### Q5: 如何导出项目数据？

**A**:
- **JSON 格式**: 导出数据结构（不含实际文件）
- **ZIP 格式**: 导出完整数据（包含所有图片和视频文件）
