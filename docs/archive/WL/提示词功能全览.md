## 1️⃣ 视觉风格提示词

### 📍 文件位置
- **核心文件**: [services/ai/promptConstants.ts](file:///Users/wl/Desktop/job/learn/WL-AI-Director/services/ai/promptConstants.ts)

### 🎨 功能说明
为不同的视觉风格提供标准化的提示词模板，确保生成的内容风格统一。

### 📦 支持的视觉风格

| 风格ID | 中文名称 | 英文提示词特点 |
|--------|---------|---------------|
| `live-action` | 真人实拍 | photorealistic, cinematic film quality, 8K resolution, film grain texture |
| `anime` | 日本动漫 | cel-shaded, vibrant saturated colors, Studio Ghibli/Makoto Shinkai quality |
| `2d-animation` | 经典2D动画 | hand-drawn style, Disney/Pixar quality, smooth clean lines |
| `3d-animation` | 3D CGI动画 | Pixar/DreamWorks style, subsurface scattering, PBR textures |
| `cyberpunk` | 赛博朋克 | neon-lit urban environment, holographic UI, volumetric fog |
| `oil-painting` | 油画风格 | visible impasto brushstrokes, classical art composition |

### 🔧 主要函数

#### `getStylePrompt(visualStyle: string)`
获取视觉风格的英文提示词，用于图像生成 API。

```typescript
const stylePrompt = getStylePrompt('anime');
// 返回: "Japanese anime style, cel-shaded, vibrant saturated colors..."
```

#### `getStylePromptCN(visualStyle: string)`
获取视觉风格的中文描述，用于 UI 显示。

```typescript
const styleDesc = getStylePromptCN('anime');
// 返回: "日本动漫风格，cel-shaded，鲜艳色彩，Studio Ghibli品质"
```

#### `getNegativePrompt(visualStyle: string)`
获取角色负面提示词，用于排除不想要的视觉元素。

```typescript
const negativePrompt = getNegativePrompt('anime');
// 返回: "photorealistic, 3d render, western cartoon, ugly, bad anatomy..."
```

#### `getSceneNegativePrompt(visualStyle: string)`
获取场景负面提示词，额外排除人物/人形元素。

```typescript
const sceneNegativePrompt = getSceneNegativePrompt('anime');
// 返回: "person, people, human, character, figure, silhouette, crowd, portrait, face, body, hands, photorealistic, 3d render..."
```

## 2️⃣ 剧本处理提示词

### 📍 文件位置
- **核心文件**: [services/ai/scriptService.ts](file:///Users/wl/Desktop/job/learn/WL-AI-Director/services/ai/scriptService.ts)

### 🎯 功能说明
处理原始剧本文本，解析为结构化数据，并生成分镜列表。

### 📝 主要功能

#### 2.1 剧本解析 (parseScriptToData)

**功能**: 将原始剧本文本解析为结构化的 JSON 数据

**提示词内容**:
```
Analyze the text and output a JSON object in the language: ${language}.

Tasks:
1. Extract title, genre, logline (in ${language}).
2. Extract characters (id, name, gender, age, personality).
3. Extract scenes (id, location, time, atmosphere).
4. Break down the story into paragraphs linked to scenes.

Output ONLY valid JSON with this structure:
{
  "title": "string",
  "genre": "string",
  "logline": "string",
  "characters": [{"id": "string", "name": "string", "gender": "string", "age": "string", "personality": "string"}],
  "scenes": [{"id": "string", "location": "string", "time": "string", "atmosphere": "string"}],
  "storyParagraphs": [{"id": number, "text": "string", "sceneRefId": "string"}]
}
```

**输出数据结构**:
- `title`: 剧本标题
- `genre`: 类型（如动作、科幻、爱情等）
- `logline`: 故事梗概
- `characters`: 角色列表（包含基本信息）
- `scenes`: 场景列表（地点、时间、氛围）
- `storyParagraphs`: 故事段落（与场景关联）

**后续处理**:
解析完成后，会自动调用以下功能：
1. 生成全局美术指导文档
2. 批量生成角色视觉提示词
3. 生成场景视觉提示词

#### 2.2 分镜生成 (generateShotList)

**功能**: 根据剧本数据和目标时长，为每个场景生成适量的分镜头

**提示词内容**:
```
Act as a professional cinematographer. Generate a detailed shot list (Camera blocking) for Scene ${index + 1}.
Language for Text Output: ${lang}.

IMPORTANT VISUAL STYLE: ${stylePrompt}
All 'visualPrompt' fields MUST describe shots in this "${visualStyle}" style.

Scene Details:
Location: ${scene.location}
Time: ${scene.time}
Atmosphere: ${scene.atmosphere}

Scene Action:
"${paragraphs.slice(0, 5000)}"

Context:
Genre: ${scriptData.genre}
Visual Style: ${visualStyle} (${stylePrompt})
Target Duration (Whole Script): ${scriptData.targetDuration || 'Standard'}
Total Shots Budget: ${totalShotsNeeded} shots (Each shot = 10 seconds of video)
Shots for This Scene: Approximately ${shotsPerScene} shots

Characters:
${JSON.stringify(scriptData.characters.map(c => ({ id: c.id, name: c.name, desc: c.visualPrompt || c.personality })))}

Professional Camera Movement Reference (Choose from these categories):
- Horizontal Left Shot (向左平移) - Camera moves left
- Horizontal Right Shot (向右平移) - Camera moves right
- Pan Left Shot (平行向左扫视) - Pan left
- Pan Right Shot (平行向右扫视) - Pan right
- Vertical Up Shot (向上直线运动) - Move up vertically
- Vertical Down Shot (向下直线运动) - Move down vertically
- Tilt Up Shot (向上仰角运动) - Tilt upward
- Tilt Down Shot (向下俯角运动) - Tilt downward
- Zoom Out Shot (镜头缩小/拉远) - Pull back/zoom out
- Zoom In Shot (镜头放大/拉近) - Push in/zoom in
- Dolly Shot (推镜头) - Dolly in/out movement
- Circular Shot (环绕拍摄) - Orbit around subject
- Over the Shoulder Shot (越肩镜头) - Over shoulder perspective
- Pan Shot (摇镜头) - Pan movement
- Low Angle Shot (仰视镜头) - Low angle view
- High Angle Shot (俯视镜头) - High angle view
- Tracking Shot (跟踪镜头) - Follow subject
- Handheld Shot (摇摄镜头) - Handheld camera
- Static Shot (静止镜头) - Fixed camera position
- POV Shot (主观视角) - Point of view
- Bird's Eye View Shot (俯瞰镜头) - Overhead view
- 360-Degree Circular Shot (360度环绕) - Full circle
- Parallel Tracking Shot (平行跟踪) - Side tracking
- Diagonal Tracking Shot (对角跟踪) - Diagonal tracking
- Rotating Shot (旋转镜头) - Rotating movement
- Slow Motion Shot (慢动作) - Slow-mo effect
- Time-Lapse Shot (延时摄影) - Time-lapse
- Canted Shot (斜视镜头) - Dutch angle
- Cinematic Dolly Zoom (电影式变焦推轨) - Vertigo effect

Instructions:
1. Create EXACTLY ${shotsPerScene} shots (or ${shotsPerScene - 1} to ${shotsPerScene + 1} shots if needed for story flow) for this scene.
2. CRITICAL: Each shot will be 10 seconds. Total shots must match the target duration formula: ${targetSeconds} seconds ÷ 10 = ${totalShotsNeeded} total shots across all scenes.
3. DO NOT exceed ${shotsPerScene + 1} shots for this scene. Select the most important moments only.
4. 'cameraMovement': Can reference the Professional Camera Movement Reference list above for inspiration, or use your own creative camera movements. You may use the exact English terms (e.g., "Dolly Shot", "Pan Right Shot", "Zoom In Shot", "Tracking Shot") or describe custom movements.
5. 'shotSize': Specify the field of view (e.g., Extreme Close-up, Medium Shot, Wide Shot).
6. 'actionSummary': Detailed description of what happens in the shot (in ${lang}).
7. 'visualPrompt': Detailed description for image generation in ${visualStyle} style (OUTPUT IN ${lang}). Include style-specific keywords. Keep it under 50 words.

Output ONLY a valid JSON OBJECT with this exact structure (no markdown, no extra text):
{
  "shots": [
    {
      "id": "string",
      "sceneId": "${scene.id}",
      "actionSummary": "string",
      "dialogue": "string (empty if none)",
      "cameraMovement": "string",
      "shotSize": "string",
      "characters": ["string"],
      "keyframes": [
        {"id": "string", "type": "start|end", "visualPrompt": "string"}
      ]
    }
  ]
}
```

**输出数据结构**:
- `shots`: 分镜数组

  - `sceneId`: 关联场景ID
  - `actionSummary`: 动作描述
  - `dialogue`: 对白（如有）
  - `cameraMovement`: 镜头运动
  - `shotSize`: 景别
  - `characters`: 涉及角色
  - `keyframes`: 关键帧数组（起始帧、结束帧）

#### 2.3 剧本续写 (continueScript)

**功能**: 基于已有剧本内容续写后续情节

**提示词内容**:
```
你是一位资深剧本创作者。请在充分理解下方已有剧本内容的基础上，续写后续情节。

续写要求：
1. 严格保持原剧本的风格、语气、人物性格和叙事节奏，确保无明显风格断层。
2. 情节发展需自然流畅，逻辑严密，因果关系合理，避免突兀转折。
3. 有效增加戏剧冲突和情感张力，使故事更具吸引力和张力。
4. 续写内容应为原有剧本长度的30%-50%，字数适中，避免过短或过长。
5. 保持剧本的原有格式，包括场景描述、人物对白、舞台指示等，确保格式一致。
6. 输出语言为：${language}，用词准确、表达流畅。

已有剧本内容：
"""
${existingScript}
"""

请开始续写：
```

**输出**: 续写的剧本文本

```
- Title: ${title}
- Genre: ${genre}
- Logline: ${logline}

- Language: ${language}

${characters.map((c, i) => `${i + 1}. ${c.name} (${c.gender}, ${c.age}, ${c.personality})`).join('\n')}

## Scenes
${scenes.map((s, i) => `${i + 1}. ${s.location} - ${s.time} - ${s.atmosphere}`).join('\n')}

## Your Task
Create a comprehensive Art Direction Brief in JSON format. This brief will be injected into EVERY subsequent visual prompt to ensure all characters and scenes share a unified look and feel.

CRITICAL RULES:
- All descriptions must be specific, concrete, and actionable for image generation AI
- The brief must define a COHESIVE visual world - characters and scenes must look like they belong to the SAME production
- Color palette must be harmonious and genre-appropriate
- Character design rules must ensure all characters share the same art style while being visually distinct from each other
- Output all descriptive text in ${language}

Output ONLY valid JSON with this exact structure:
{
  "colorPalette": {
    "primary": "primary color tone description (e.g., 'deep navy blue with slight purple undertones')",
    "secondary": "secondary color description",
    "accent": "accent/highlight color",
    "skinTones": "skin tone range for characters in this style (e.g., 'warm ivory to golden tan, with soft peach undertones')",
    "saturation": "overall saturation tendency (e.g., 'medium-high, slightly desaturated for cinematic feel')",
    "temperature": "overall color temperature (e.g., 'cool-leaning with warm accent lighting')"
  },
  "characterDesignRules": {
    "proportions": "body proportion style (e.g., '7.5 head-to-body ratio, athletic builds, realistic proportions' or '6 head ratio, stylized anime proportions')",
    "eyeStyle": "unified eye rendering approach (e.g., 'large expressive anime eyes with detailed iris reflections' or 'realistic eye proportions with cinematic catchlights')",
    "lineWeight": "line/edge style (e.g., 'clean sharp outlines with 2px weight' or 'soft edges with no visible outlines, photorealistic blending')",
    "detailLevel": "detail density (e.g., 'high detail on faces and hands, medium on clothing textures, stylized backgrounds')"
  },
  "lightingStyle": "unified lighting approach (e.g., 'three-point cinematic lighting with strong rim light, warm key light from 45-degree angle, cool fill')",

This paragraph will be prepended to every character and scene prompt to anchor visual style. It should describe: overall rendering quality, specific art style fingerprint, color grading approach, lighting philosophy, and emotional tone of visuals. Write it as direct instructions to an image generation AI."
}
```

**输出数据结构**:
- `colorPalette`: 色彩配置
  - `primary`: 主色调
  - `secondary`: 次要色调
  - `accent`: 强调色
  - `skinTones`: 皮肤色调范围
  - `saturation`: 饱和度
  - `temperature`: 色温
- `characterDesignRules`: 角色设计规则
  - `proportions`: 身体比例
  - `eyeStyle`: 眼睛风格

- `lightingStyle`: 光照风格

- `moodKeywords`: 情绪关键词数组

- **角色细节**：表情、姿态、服装、毛发
- **环境细节**：氛围、透视、叙事元素
- **运动暗示**：动态模糊、视觉引导

#### 8.4 负面提示词使用
- 角色生成：使用 `getNegativePrompt`
- 场景生成：使用 `getSceneNegativePrompt`
- 排除不想要的视觉元素

### 🔧 常见问题

#### Q1: 提示词生成失败怎么办？
**A**: 系统会自动降级到基础模板，不会影响用户体验。可以检查：
- API Key 是否有效
- 网络连接是否正常
- 模型是否可用

#### Q2: 如何提高生成质量？
**A**:
- 先完善叙事动作描述
- 使用 AI 优化关键帧
- 使用 AI 生成动作建议
- 手动微调生成的提示词

#### Q3: 不同视觉风格的提示词有什么区别？
**A**: 每种风格都有独特的提示词关键词：
- **live-action**: photorealistic, 8K, film grain
- **anime**: cel-shaded, vibrant colors, Studio Ghibli
- **3d-animation**: PBR textures, subsurface scattering
- **cyberpunk**: neon, holographic, volumetric fog

#### Q4: 如何确保角色一致性？
**A**:
- 使用角色九宫格造型设计
- 生成时参考 turnaround sheet
- 使用相同的美术指导文档
- 保持相同的视觉风格

## 📊 总结

WL-AI-Director 项目中的提示词系统涵盖了从剧本解析到视频生成的完整流程，主要分为以下几类：

| 类别 | 功能 | 文件位置 |
|------|------|---------|
| **视觉风格** | 6种视觉风格的标准化提示词 | `services/ai/promptConstants.ts` |
| **剧本处理** | 剧本解析、分镜生成、续写改写 | `services/ai/scriptService.ts` |
| **视觉资产** | 美术指导、角色/场景/道具视觉提示词 | `services/ai/visualService.ts` |
| **分镜制作** | 关键帧优化、动作生成、镜头拆分 | `services/ai/shotService.ts` |
| **视频生成** | 视频生成提示词构建 | `services/ai/videoService.ts` |
| **UI编辑** | 提示词编辑和管理界面 | `components/StagePrompts/` |

所有提示词都遵循以下原则：
- ✅ 风格一致性
- ✅ 细节丰富
- ✅ 可操作性强
- ✅ 支持AI增强
- ✅ 用户可编辑
