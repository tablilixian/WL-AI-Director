# 图片处理服务整合计划

## 概述

本计划将项目中分散的图片处理功能整合到统一的服务中，解决代码重复和 CORS 兼容性问题。

---

## 阶段一：创建统一图片服务

### 1.1 创建 unifiedImageService.ts

**目标**: 创建 `services/unifiedImageService.ts`，提供全面的图片处理能力

**包含功能**:

| 方法 | 功能 | 说明 |
|------|------|------|
| `parseUrl(url)` | 解析 URL 类型 | 返回 ImageSource |
| `resolveForDisplay(url)` | 解析为显示用 URL | 返回 blob: 或 data: |
| `resolveForApi(url)` | 解析为 API 用 Base64 | 返回 data:base64 |
| `getDimensions(url)` | 获取图片尺寸 | 兼容 CORS |
| `saveImage(id, blob)` | 保存到本地 | 封装 imageStorageService |
| `getImage(id)` | 从本地读取 | 返回 Blob |
| `deleteImage(id)` | 删除本地图片 | - |
| `generateImageId()` | 生成图片 ID | - |
| `blobToBase64(blob)` | Blob 转 Base64 | - |
| `base64ToBlob(base64)` | Base64 转 Blob | - |

**实现要点**:
- `getDimensions` 必须兼容所有 URL 格式（data:/blob:/local:/http:/https:/）
- 网络 URL 使用 fetch + createObjectURL 绕过 CORS
- 内部复用 `imageStorageService` 的底层存储能力

## 代码规范

### 注释要求

每个导出的方法必须包含以下注释:

```typescript
/**
 * 方法功能描述
 * 
 * @param paramName - 参数说明
 * @returns 返回值说明
 * @throws 可能抛出的错误
 * 
 * @example
 * // 使用示例
 * const url = await resolveForDisplay('local:img_123');
 */
```

### 错误处理

- 所有异步方法必须有 try-catch
- 错误需要记录日志
- 用户可见的错误需要抛出有意义的提示
