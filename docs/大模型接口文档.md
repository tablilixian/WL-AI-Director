## 🏢 支持的模型提供商

- **提供商ID**: `antsk`
- **基础URL**: `https://api.antsk.cn`

- **认证方式**: Bearer Token

## 💬 对话模型接口

### 接口概述

对话模型用于剧本解析、分镜生成、关键帧优化、动作生成等文本生成任务。

### 核心文件

- **API核心**: `services/ai/apiCore.ts`
- **适配器**: `services/adapters/chatAdapter.ts`
- **业务服务**: 
  - `services/ai/scriptService.ts` - 剧本处理
  - `services/ai/shotService.ts` - 分镜辅助
  - `services/ai/visualService.ts` - 视觉资产生成

### 1. Chat Completion API（非流式）

#### 接口信息

| 项目 | 内容 |
|------|------|
| **端点** | `/v1/chat/completions` |
| **方法** | POST |
| **超时** | 600000ms (10分钟) |
| **认证** | Bearer Token |

#### 请求参数

```typescript
{
  model: string,              // 模型名称，如 "gpt-5.1", "glm-4-flash"
  messages: Array<{
    role: 'user' | 'system' | 'assistant',
    content: string | Array<{
      type: 'text' | 'image_url',
      text?: string,
      image_url?: { url: string }
    }>
  }>,
  temperature: number,        // 0-2，默认 0.7
  max_tokens?: number,        // 最大token数，可选
  response_format?: {         // 响应格式，可选
    type: 'json_object'
  }
}
```

#### 响应格式

```typescript
{
  choices: Array<{
    message: {
      role: 'assistant',
      content: string
    },
    finish_reason: string
  }>,
  usage?: {
    prompt_tokens: number,
    completion_tokens: number,
    total_tokens: number
  }
}
```

#### 调用示例

```typescript
// 基础调用
const response = await chatCompletion(
  "请分析这段剧本...",
  "gpt-5.1",           // 模型ID
  0.7,                 // temperature
  8192,                // maxTokens
  'json_object'        // responseFormat
);

// 带图片的调用（用于视频生成）
const messages = [
  {
    role: 'user',
    content: [
      { type: 'text', text: '生成视频...' },
      { type: 'image_url', image_url: { url: 'data:image/png;base64,...' } }
    ]
  }
];
```

### 2. Chat Completion Stream API（流式）

#### 接口信息

| 项目 | 内容 |
|------|------|
| **端点** | `/v1/chat/completions` |
| **方法** | POST |
| **超时** | 600000ms (10分钟) |
| **认证** | Bearer Token |
| **特点** | SSE (Server-Sent Events) 流式响应 |

#### 请求参数

```typescript
{
  model: string,
  messages: Array<{ role: string, content: string }>,
  temperature: number,
  stream: true,              // 启用流式
  response_format?: {
    type: 'json_object'
  }
}
```

#### 响应格式（SSE）

```
data: {"choices":[{"delta":{"content":"文本内容"}}]}

data: {"choices":[{"delta":{"content":"更多内容"}}]}

data: [DONE]
```

#### 调用示例

```typescript
const fullText = await chatCompletionStream(
  "生成长文本...",
  "gpt-5.1",
  0.7,
  undefined,
  600000,
  (delta) => {
    console.log('收到增量:', delta);
  }
);
```

### 1. Gemini Image API

#### 接口信息

| 项目 | 内容 |
|------|------|
| **提供商** | AntSK |
| **端点** | `/v1beta/models/{model}:generateContent` |
| **方法** | POST |
| **超时** | 600000ms (10分钟) |
| **认证** | Bearer Token |

#### 请求参数

```typescript
{
  contents: [{
    role: 'user',
    parts: Array<{
      text?: string,           // 提示词
      inlineData?: {           // 图片数据（参考图）
        mimeType: 'image/png' | 'image/jpeg',
        data: string           // base64编码
      }
    }>
  }],
  generationConfig: {
    responseModalities: ['TEXT', 'IMAGE'],
    imageConfig?: {
      aspectRatio: '16:9' | '9:16' | '1:1'
    }
  }
}
```

#### 响应格式

```typescript
{
  candidates: [{
    content: {
      parts: [{
        inlineData?: {
          mimeType: 'image/png',
          data: string           // base64编码的图片
        }
      }]
    }
  }]
}
```

#### 调用示例

```typescript
const imageUrl = await callImageApi({
  prompt: "生成一个动漫风格的角色...",
  referenceImages: ['data:image/png;base64,...'],  // 参考图
  aspectRatio: '16:9',
  resourceType: 'character',
  resourceId: 'char-001'
});
```

## 🔧 API Key 管理

### 1. API Key 存储位置

- **全局 API Key**: `localStorage` 键名 `antsk_api_key`
- **模型专属 API Key**: 存储在模型注册中心（`bigbanana_model_registry`）
- **提供商 API Key**: 存储在提供商配置中

### 2. API Key 获取优先级

1. 模型专属 API Key（`model.apiKey`）

## ⚠️ 错误处理

### 1. 错误类型

#### ApiKeyError

```typescript
class ApiKeyError extends Error {
  name: 'ApiKeyError';
  message: string;
}
```

**触发场景**: API Key 缺失或无效

#### HTTP 错误

| 状态码 | 含义 | 处理方式 |
|--------|------|----------|
| 400 | 客户端错误 | 不重试，直接返回错误信息 |
| 401 | 认证失败 | 不重试，提示 API Key 错误 |
| 403 | 权限不足 | 不重试，提示权限问题 |
| 429 | 限流 | 自动重试（指数退避） |
| 500 | 服务器错误 | 自动重试（指数退避） |
| 504 | 网关超时 | 自动重试（指数退避） |

### 2. 错误响应格式

```typescript
{
  error: {
    message: string,
    code?: string,
    type?: string
  }
}
```

### 3. 常见错误消息

| 错误消息 | 原因 | 解决方案 |
|----------|------|----------|
| "API Key 缺失，请在模型配置中设置 API Key" | 未配置 API Key | 在设置页面配置 API Key |
| "提示词可能包含不安全或违规内容" | 内容审核未通过 | 修改提示词，避免敏感词汇 |
| "当前请求较多，暂时未能处理成功" | 服务器繁忙 | 稍后重试 |
| "请求超时" | 网络或服务器响应慢 | 检查网络，稍后重试 |
| "视频生成超时 (20分钟)" | 视频生成时间过长 | 稍后重试或联系支持 |

## 🔄 重试机制

### 1. 重试策略

**服务文件**: `services/ai/apiCore.ts`

#### retryOperation

**参数**:
```typescript
{
  operation: () => Promise<T>,  // 要执行的操作
  maxRetries: number,           // 最大重试次数，默认 3
  baseDelay: number             // 基础延迟（毫秒），默认 2000
}
```

**指数退避算法**:
- 第 1 次重试: 2000ms
- 第 2 次重试: 4000ms
- 第 3 次重试: 8000ms

### 2. 可重试的错误

- HTTP 429（限流）
- HTTP 500（服务器错误）
- HTTP 504（网关超时）
- 网络错误（ECONNRESET, ETIMEDOUT）
- 包含特定关键词的错误：
  - 'quota'
  - 'RESOURCE_EXHAUSTED'
  - '超时'
  - 'timeout'
  - 'Gateway Timeout'

### 3. 不可重试的错误

- HTTP 400（客户端错误）
- HTTP 401（认证失败）
- HTTP 403（权限不足）
- 包含敏感内容错误
