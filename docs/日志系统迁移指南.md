# 日志系统迁移指南

## 概述

项目已统一使用 `services/logger.ts` 提供的日志服务，替代原有的 `console.log/warn/error/info/debug` 调用。

## 为什么需要迁移？

1. **统一管理**：所有日志通过统一接口输出，便于后续管理和分析
2. **级别控制**：支持 DEBUG、INFO、WARN、ERROR 四个级别，可动态控制
3. **分类管理**：支持按功能模块分类（APP、AUTH、STORAGE、API、AI 等）
4. **持久化**：支持将日志存储到本地，便于问题追踪
5. **性能优化**：可根据需要关闭特定分类的日志，减少性能开销

## 快速开始

### 1. 导入日志服务

```typescript
import { logger, LogCategory, LogLevel } from './services/logger';
```

### 2. 替换 console 调用

#### 原代码：
```typescript
console.log('用户登录成功');
console.warn('API Key 缺失');
console.error('保存失败', error);
console.debug('调试信息', data);
```

#### 新代码：
```typescript
logger.info(LogCategory.AUTH, '用户登录成功');
logger.warn(LogCategory.API, 'API Key 缺失');
logger.error(LogCategory.STORAGE, '保存失败', error);
logger.debug(LogCategory.APP, '调试信息', data);
```

## 日志分类 (LogCategory)

| 分类 | 说明 | 使用场景 |
|--------|--------|----------|
| `APP` | 应用核心逻辑 | 项目加载、保存、状态变更等 |
| `AUTH` | 认证相关 | 登录、注册、登出等 |
| `STORAGE` | 存储相关 | IndexedDB、LocalStorage 操作等 |
| `API` | API 调用 | 网络请求、API 响应等 |
| `AI` | AI 服务 | 文本生成、图片生成、视频生成等 |
| `VIDEO` | 视频处理 | 视频合并、播放、导出等 |
| `IMAGE` | 图片处理 | 图片加载、缩略图生成等 |
| `RENDER` | 渲染相关 | 渲染日志、渲染进度等 |
| `MODEL` | 模型配置 | 模型选择、参数配置等 |
| `UI` | 用户界面 | 组件交互、事件处理等 |
| `NETWORK` | 网络相关 | 网络状态、连接问题等 |

## 日志级别 (LogLevel)

| 级别 | 值 | 说明 | 使用场景 |
|--------|------|--------|----------|
| `DEBUG` | 0 | 调试信息 | 详细的执行流程、变量值等 |
| `INFO` | 1 | 一般信息 | 正常的业务流程、状态变更等 |
| `WARN` | 2 | 警告信息 | 潜在问题、降级处理等 |
| `ERROR` | 3 | 错误信息 | 异常、失败操作等 |
| `NONE` | 4 | 禁用所有日志 | 生产环境等 |

## 高级用法

### 动态控制日志级别

```typescript
// 设置最小日志级别（低于此级别的日志不会输出）
logger.setMinLevel(LogLevel.INFO); // 只输出 INFO、WARN、ERROR

// 设置为 DEBUG 输出所有日志
logger.setMinLevel(LogLevel.DEBUG);

// 禁用所有日志
logger.setMinLevel(LogLevel.NONE);
```

### 控制日志输出目标

```typescript
// 启用/禁用控制台输出
logger.setEnableConsole(true);  // 启用（默认）
logger.setEnableConsole(false); // 禁用

// 启用/禁用本地存储
logger.setEnableStorage(true);  // 启用存储
logger.setEnableStorage(false); // 禁用存储（默认）
```

### 控制特定分类的日志

```typescript
// 启用特定分类
logger.setCategoryEnabled(LogCategory.AI, true);

// 禁用特定分类
logger.setCategoryEnabled(LogCategory.DEBUG, false);
```

### 获取和导出日志

```typescript
// 获取所有存储的日志
const logs = logger.getStorage();

// 导出日志为 JSON
const logsJson = logger.exportStorage();

// 清空存储的日志
logger.clearStorage();
```

### 监听日志事件

```typescript
// 添加监听器
const unsubscribe = logger.addListener((entry) => {
  console.log('新日志:', entry);
});

// 移除监听器
unsubscribe();
```

## 最佳实践

1. **选择合适的分类**：根据日志内容选择最相关的分类
2. **选择合适的级别**：
   - DEBUG：详细的调试信息
   - INFO：正常的业务流程
   - WARN：潜在问题或降级处理
   - ERROR：异常和失败操作
3. **避免敏感信息**：不要记录密码、密钥等敏感信息
4. **使用有意义的消息**：日志消息应该清晰、简洁、有意义
5. **合理使用数据参数**：将结构化数据作为第二个参数传递

## 配置持久化

日志配置会自动保存到 `localStorage` 的 `logger_config` 键中，包括：
- `minLevel`: 最小日志级别
- `enableConsole`: 是否启用控制台输出
- `enableStorage`: 是否启用本地存储
- `categories`: 启用的分类列表

## 性能考虑

- 日志存储默认最多保留 1000 条
- 可以通过 `logger.setCategoryEnabled()` 禁用不需要的分类
- 生产环境建议设置 `LogLevel.INFO` 或更高

## 常见问题

### Q: 为什么要替换 console？
A: console 调用分散在项目各处，难以统一管理和控制。使用统一的日志服务可以：
- 集中控制日志级别
- 按分类过滤日志
- 持久化日志用于问题追踪
- 为后续添加日志分析功能做准备

### Q: 旧的 console 调用会立即报错吗？
A: 不会。旧的 console 调用仍然可以正常工作，但建议逐步迁移到新的日志服务。

### Q: 如何在开发环境启用所有日志？
A: 在应用初始化时调用：
```typescript
logger.setMinLevel(LogLevel.DEBUG);
logger.setEnableStorage(true);
```

### Q: 如何在生产环境禁用 DEBUG 日志？
A: 在应用初始化时调用：
```typescript
logger.setMinLevel(LogLevel.INFO);
```
