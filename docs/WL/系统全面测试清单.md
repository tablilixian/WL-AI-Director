## 测试概述

### 测试目标

- ✅ 验证所有核心功能是否正常工作
- ✅ 检查用户界面的易用性
- ✅ 验证数据存储和同步的准确性
- ✅ 测试边界条件和异常处理
- ✅ 确保系统稳定性和性能

### 测试范围

- **前端功能**: React 组件、用户交互、状态管理
- **后端功能**: API 调用、数据处理、错误处理
- **数据存储**: IndexedDB 本地存储、Supabase 云端存储
- **AI 功能**: 剧本解析、图片生成、视频生成
- **集成测试**: 端到端工作流程

### 测试类型

| 类型 | 说明 | 优先级 |
|------|------|--------|
| 功能测试 | 验证每个功能是否按预期工作 | 高 |
| 界面测试 | 验证用户界面的正确性和美观性 | 中 |
| 数据测试 | 验证数据的存储、读取、同步 | 高 |
| 性能测试 | 验证系统响应速度和资源使用 | 中 |
| 兼容性测试 | 验证不同浏览器和设备的兼容性 | 低 |

## 测试环境准备

### 环境要求

- **操作系统**: macOS / Windows / Linux
- **浏览器**: Chrome / Firefox / Safari / Edge
- **Node.js**: >= 18.0.0
- **数据库**: IndexedDB (浏览器内置) + Supabase

### 前置条件

- [ ] 已安装 Node.js 和 npm
- [ ] 已克隆项目代码
- [ ] 已安装项目依赖 (`npm install`)
- [ ] 已配置 Supabase 账号
- [ ] 已获取 API Keys (OpenAI / Google / 其他)
- [ ] 已启动开发服务器 (`npm run dev`)
- [ ] 已清空本地数据库（用于干净测试）

### 测试数据准备

- [ ] 准备测试用剧本文本
- [ ] 准备测试用图片（角色、场景、道具）
- [ ] 准备测试用视频片段
- [ ] 准备测试用 API Keys
