## 文档使用指南

### 查阅项目状态
1. 查看 `00-项目总览.md` 了解整体情况
2. 查看 `01-功能模块.md` 了解具体功能状态
3. 查看 `04-更新日志.md` 了解最新更新

### 了解技术架构
1. 查看 `02-技术架构.md` 了解整体架构
2. 查看具体模块的架构说明
3. 参考数据流和部署架构

### 开始开发
1. 查看 `03-开发指南.md` 了解开发环境配置
2. 参考开发规范和代码风格
3. 查看调试技巧和测试方法

### 更新文档
1. 根据更新内容修改对应文档
2. 保持文档格式一致
3. 更新文档的"最后更新时间"
4. 在 `04-更新日志.md` 中记录更新

## 文档维护规范

### 更新频率
- **00-项目总览.md**: 每次大版本更新
- **01-功能模块.md**: 每次功能更新
- **02-技术架构.md**: 架构变更时
- **03-开发指南.md**: 开发流程变更时
- **04-更新日志.md**: 每次提交更新

### 文档格式
- 使用 Markdown 格式
- 使用清晰的章节结构
- 使用 emoji 图标增强可读性
- 保持一致的更新风格

### 更新流程
1. 修改对应文档内容
2. 更新"最后更新时间"
3. 在 `04-更新日志.md` 中记录
4. 提交到 Git
