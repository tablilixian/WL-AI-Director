## 1. 概述

角色与场景管理模块是 WL AI Director 的核心功能之一，用于管理项目中的角色、场景和道具资源。该模块支持 AI 自动生成图片、本地图片上传、资产库管理等功能。

### 核心特性
- ✅ AI 自动生成角色/场景/道具图片
- ✅ 本地图片上传支持
- ✅ 角色变体管理（服装变化）
- ✅ 角色九宫格造型生成
- ✅ 批量生成功能
- ✅ 资产库管理（本地 IndexedDB + 云端 Supabase）
- ✅ 图片预览功能
- ✅ 横竖屏切换
- ✅ 自动检测和修复卡住的生成状态

## 2. 角色管理

### 2.1 新建角色

**入口**：角色库区域 → "新建角色" 按钮

**功能说明**：
- 点击按钮后创建一个新的角色对象
- 默认信息：
  - 名称：新角色
  - 性别：未设定
  - 年龄：未设定
  - 性格：待补充
  - 状态：pending（待生成）
- 自动生成唯一 ID（格式：char_xxx）

**操作流程**：
1. 点击"新建角色"按钮
2. 系统创建新角色卡片
3. 编辑角色基本信息
4. 编辑角色提示词
5. 生成或上传角色图片

---

### 2.2 删除角色

**入口**：角色卡片 → "删除角色" 按钮

**功能说明**：
- 删除指定角色及其所有变体
- 删除前会弹出确认对话框
- 警告信息：删除角色会影响所有使用该角色的分镜

**操作流程**：
1. 点击角色卡片的"删除角色"按钮
2. 确认删除对话框
3. 系统删除角色数据
4. 更新项目数据

---

### 2.3 编辑角色信息

**入口**：角色卡片 → 点击名称/性别/年龄/性格字段

**功能说明**：
- 支持内联编辑角色基本信息
- 可编辑字段：
  - 名称（必填）
  - 性别（必填）
  - 年龄（必填）
  - 性格（必填）

**操作流程**：
1. 点击要编辑的字段
2. 输入新内容
3. 按 Enter 键或点击外部保存
4. 系统自动更新项目数据

---

### 2.4 编辑角色提示词

**入口**：角色卡片 → "角色提示词" 区域

**功能说明**：
- 支持多行文本输入
- 最大高度限制：160px
- 自动保存到项目数据
- 提示词用于 AI 生成角色图片

**操作流程**：
1. 点击提示词编辑区域
2. 输入或修改提示词内容
3. 点击"保存"按钮
4. 系统自动更新项目数据

---

### 2.5 生成角色图片

**入口**：角色卡片 → "生成" 按钮

**功能说明**：
- 使用 AI 根据角色提示词生成图片
- 支持横竖屏切换（16:9 / 9:16 / 1:1）
- 生成状态：
  - pending：待生成
  - generating：生成中（显示加载动画）
  - completed：生成完成
  - failed：生成失败

**操作流程**：
1. 编辑角色提示词
2. 选择横竖屏（可选）
3. 点击"生成"按钮
4. 等待 AI 生成图片
5. 生成完成后显示图片

**技术细节**：
- 图片保存到本地 IndexedDB
- 返回格式：`local:{image_id}`
- 设置 `referenceImageSource = 'local'`
- 设置 `localImageId = {image_id}`

---

### 2.6 上传角色图片

**入口**：角色卡片 → "上传" 按钮

**功能说明**：
- 支持本地图片上传
- 支持的格式：JPG、PNG、GIF、WEBP
- 文件大小限制：10MB
- 自动保存到本地 IndexedDB

**操作流程**：
1. 点击"上传"按钮
2. 选择本地图片文件
3. 系统自动上传到 IndexedDB
4. 显示上传的图片

**技术细节**：
- 图片保存到本地 IndexedDB
- 返回格式：`local:{image_id}`
- 设置 `referenceImageSource = 'local'`
- 设置 `localImageId = {image_id}`

### 2.7 角色变体管理

#### 2.7.1 打开衣柜

**入口**：角色卡片 → "衣柜" 按钮

**功能说明**：
- 显示角色的所有变体（服装变化）
- 支持添加新变体
- 支持删除变体
- 支持生成变体图片
- 支持上传变体图片

**操作流程**：
1. 点击"衣柜"按钮
2. 打开衣柜模态框
3. 查看现有变体
4. 添加/删除/生成变体

---

#### 2.7.2 添加变体

**入口**：衣柜模态框 → "添加变体" 区域

**功能说明**：
- 创建新的角色变体
- 需要输入：
  - 变体名称（必填）
  - 变体提示词（必填）
- 自动生成唯一 ID（格式：var_xxx）

**操作流程**：
1. 输入变体名称
2. 输入变体提示词
3. 点击"添加变体"按钮
4. 系统创建新变体
5. 生成或上传变体图片

---

#### 2.7.3 生成变体图片

**入口**：衣柜模态框 → 变体卡片 → "生成" 按钮

**功能说明**：
- 使用 AI 根据变体提示词生成图片
- 保持角色基础外观一致
- 只改变服装/造型

**操作流程**：
1. 编辑变体提示词
2. 点击"生成"按钮
3. 等待 AI 生成图片
4. 生成完成后显示图片

**技术细节**：
- 使用角色主图作为参考
- 确保面部和身份一致性
- 只改变服装/造型

---

#### 2.7.4 上传变体图片

**入口**：衣柜模态框 → 变体卡片 → "上传" 按钮

**功能说明**：
- 支持本地图片上传
- 支持的格式：JPG、PNG、GIF、WEBP
- 文件大小限制：10MB

**操作流程**：
1. 点击"上传"按钮
2. 选择本地图片文件
3. 系统自动上传到 IndexedDB
4. 显示上传的图片

---

#### 2.7.5 删除变体

**入口**：衣柜模态框 → 变体卡片 → "删除" 按钮

**功能说明**：
- 删除指定变体
- 删除前会弹出确认对话框

**操作流程**：
1. 点击"删除"按钮
2. 确认删除对话框
3. 系统删除变体数据

### 2.8 角色九宫格造型

#### 2.8.1 打开九宫格

**入口**：角色卡片 → "九宫格" 按钮

**功能说明**：
- 显示角色的九宫格造型图
- 支持生成九宫格
- 支持编辑视角描述
- 支持重新生成图片
- 支持加入资产库

**操作流程**：
1. 点击"九宫格"按钮
2. 打开九宫格模态框
3. 查看或生成九宫格

---

#### 2.8.2 生成九宫格视角描述

**入口**：九宫格模态框 → "生成视角描述" 按钮

**功能说明**：
- 使用 AI 生成 9 个视角的描述
- 视角包括：
  1. 正面
  2. 正面 45°
  3. 侧面
  4. 背面 45°
  5. 背面
  6. 背面 45°（另一侧）
  7. 侧面（另一侧）
  8. 正面 45°（另一侧）
  9. 俯视

**操作流程**：
1. 点击"生成视角描述"按钮
2. 等待 AI 生成视角描述
3. 查看生成的 9 个视角描述
4. 可手动编辑每个视角的描述

---

#### 2.8.3 生成九宫格图片

**入口**：九宫格模态框 → "生成九宫格" 按钮

**功能说明**：
- 使用 AI 根据视角描述生成 9 张图片
- 合成一张九宫格造型图
- 保持角色外观一致性

**操作流程**：
1. 确认或编辑视角描述
2. 点击"生成九宫格"按钮
3. 等待 AI 生成 9 张图片
4. 系统自动合成九宫格
5. 显示九宫格图片

**技术细节**：
- 图片保存到本地 IndexedDB
- 返回格式：`local:{image_id}`
- 设置 `imageUrlSource = 'local'`
- 设置 `localImageId = {image_id}`

---

#### 2.8.4 重新生成九宫格图片

**入口**：九宫格模态框 → "重新生成图片" 按钮

**功能说明**：
- 保留已有的视角描述
- 仅重新生成九宫格图片
- 适用于对文案满意但图片效果不好的情况

**操作流程**：
1. 点击"重新生成图片"按钮
2. 等待 AI 重新生成图片
3. 显示新生成的九宫格

---

#### 2.8.5 编辑视角描述

**入口**：九宫格模态框 → 点击某个视角

**功能说明**：
- 支持手动编辑每个视角的描述
- 可编辑字段：
  - 视角（viewAngle）
  - 景别（shotSize）
  - 描述（description）

**操作流程**：
1. 点击要编辑的视角
2. 修改视角描述
3. 点击"保存"按钮
4. 系统更新视角数据

---

### 2.9 加入资产库

**入口**：角色卡片 → "加入资产库" 按钮

**功能说明**：
- 将角色保存到资产库
- 如果图片是本地的，先上传到云端
- 更新项目数据为云端 URL
- 保存到本地 IndexedDB

**操作流程**：
1. 点击"加入资产库"按钮
2. 如果图片是本地的，上传到云端
3. 系统创建资产库项目
4. 显示成功提示

**技术细节**：
- 云端路径：`{user_id}/asset_library/character/{char_id}.png`
- 更新 `referenceImage` 为云端 URL
- 更新 `referenceImageSource` 为 'cloud'
- 清除 `localImageId`
- 保存到本地 IndexedDB（资产库）

---

### 2.10 从资产库选择

**入口**：角色库区域 → "从资产库选择" 按钮

**功能说明**：
- 打开资产库模态框
- 显示所有已保存的角色
- 支持搜索和过滤
- 支持替换现有角色

**操作流程**：
1. 点击"从资产库选择"按钮
2. 打开资产库模态框
3. 搜索或过滤角色
4. 选择要使用的角色
5. 确认导入

## 7. 图片存储机制

### 7.1 本地存储（IndexedDB）

**用途**：
- 存储用户上传的图片
- 存储 AI 生成的图片
- 存储资产库元数据

**特点**：
- 快速访问
- 离线可用
- 自动清理过期图片（默认 7 天）

**存储格式**：
- 图片 ID 格式：`img_{timestamp}_{random}`
- 引用格式：`local:{image_id}`
- 数据结构：
  ```typescript
  {
    id: string;
    blob: Blob;
    createdAt: number;
  }
  ```

---

### 7.2 云端存储（Supabase）

**用途**：
- 存储加入资产库的图片
- 存储项目配置 JSON

**特点**：
- 跨设备同步
- 持久化存储
- 用户隔离

**存储路径**：
- 角色：`{user_id}/asset_library/character/{char_id}.png`
- 场景：`{user_id}/asset_library/scene/{scene_id}.png`
- 道具：`{user_id}/asset_library/prop/{prop_id}.png`
- 九宫格：`{user_id}/asset_library/turnaround/{char_id}.png`

---

### 7.3 图片源标识

**local**：
- 图片存储在本地 IndexedDB
- 引用格式：`local:{image_id}`
- 需要使用 `useImageLoader` hook 加载

**cloud**：
- 图片存储在云端 Supabase
- 引用格式：`https://...`
- 可以直接作为 `img` 标签的 `src`

**base64**：
- 图片以 base64 格式直接嵌入
- 引用格式：`data:image/png;base64,...`
- 不推荐（文件过大）

---

## 15. 常见问题

### Q1：为什么生成的图片不显示？

**A**：可能的原因：
1. 图片正在加载中（显示加载动画）
2. 图片生成失败（显示错误提示）
3. 本地图片读取失败（检查 IndexedDB）

**解决方法**：
1. 等待加载完成
2. 查看错误提示，检查 API Key
3. 刷新页面重试

---

### Q2：如何确保角色一致性？

**A**：推荐做法：
1. 使用九宫格造型生成多个视角
2. 在分镜生成时使用九宫格作为参考
3. 使用变体管理不同服装，保持面部一致

---

### Q3：本地图片和云端图片有什么区别？

**A**：
- **本地图片**：存储在 IndexedDB，格式为 `local:{id}`，需要使用 `useImageLoader` 加载
- **云端图片**：存储在 Supabase，格式为 `https://...`，可以直接使用
- **推荐**：生成时保存到本地，加入资产库时上传到云端

---

### Q4：如何删除本地缓存的图片？

**A**：
- 系统会自动清理 7 天前的图片
- 可以通过浏览器开发者工具手动清理 IndexedDB
- 删除项目时会清理相关图片

---

### Q5：批量生成会消耗大量 API 配额吗？

**A**：
- 是的，批量生成会逐个调用 AI API
- 建议先编辑好所有提示词，再批量生成
- 可以使用"重新生成"功能，避免浪费配额
